# ②-A / spec-A7 登録フロー（入り口・統一ダイアログ・ガイダンス）

> ステータス: 清書（2026-07-06）
> 位置づけ: ②-A の登録フロー分冊。全ジャンルの「記録の入り口」を洗い出し、**どのジャンルも同じ形**に揃え、マイクロコピー／オンボーディングで**迷わせず・負担を減らす**登録体験を定める正本。
> 関連: 08（各テンプレ確定構成）／ spec-A1（Event/Visit・スナップショット・自動フィルの格納先）／ docs/11（自動取得のデータ源）／ favoreco-feasibility §4-1e/§4-1f
>
> 設計原則（ルートCLAUDE.md §3）: 主要アクションに2タップ以内・存在＝入力済み・失敗しない設計。

---

## 1. フックの型（記録を始める"入り口"）

| 記号 | フック | 例 |
|---|---|---|
| 🔍 | 検索サジェスト（外部メタAPI） | 映画=TMDb・書籍=タイトル→openBD |
| 📷 | スキャン／写真OCR | 書籍=ISBNバーコード・酒=ラベル・書籍=奥付 |
| 📍 | 場所・マスター選択（Apple POI／参照DB） | 施設・御朱印の寺社城港・会場・酒蔵 |
| 🔗 | URL貼付→OGP | 映画=映画.com/公式・観劇=チケサイト/公式X |
| ✍️ | 手動 | 全ジャンル共通の最終フォールバック |

### 主フック / 副フック
- **主フック**＝そのジャンルで**最も自然で・最も自動で埋まる既定の入り口**（画面で目立たせる）。
- **副フック**＝主が使えない/別ルートが良いときの**代わり**（保険・控えめに配置）。
- **原則**＝「まず主、ダメなら副、最後は必ず✍️手動」。どのフックが無くても詰まらない。

> **観察**：映画・書籍・酒＝外部メタ自動フィル（TMDb/NDL・openBD/さけのわ）／施設・御朱印・会場系＝場所フック（Apple POI）／観劇・美術展・ライブ＝公開DBは無いが **URL(OGP)を主フックに格上げ**して"最低限"（公演名・キービジュアル・概要）を自動取得（2026-07-06 ユーザー指示）。OGPの質はサイト依存だが、確実に取れるのは og:title / og:image / og:description ＝タイトル・ビジュアル・概要。構造化データ（キャスト/セトリ）は手動。**全て §3 の同じ骨格に畳める**。

## 3. 統一登録フロー（全ジャンル共通の1つの形）

> **① 探す/撮る/選ぶ → ② 候補 → ③ 自動フィルをプレビュー確認 → ④ 自分の記録を足す**

```
┌─────────────────────────────┐
│ ＋ 新規記録：映画                              │
│ ┌───────────────────────────┐ │
│ │ 🔍 タイトルで探す（例：ひつじ探偵団）        │ │ ← スマート入力バー（常に最上部・同じ位置）
│ └───────────────────────────┘ │
│  [📷 スキャン] [🔗 URL] [📍 場所] [✍️ 手動]     │ ← 使えるフックだけ点灯（主を強調・副は控えめ）
│ ─────────────────────────────  │
│ 候補：🎬ひつじ探偵団 (2026) [poster]            │ ← 視覚で選ぶ（年・サムネで区別）
│ ─────────────────────────────  │
│ ✅ 自動で入りました。違ったら直せます             │ ← プレビュー（勝手に確定しない）
│    原題/監督/出演者/あらすじ…                   │
│ ＋ あなたの記録：⭐評価 / 鑑賞日 / 感想（後回しOK） │ ← 自分の分
└─────────────────────────────┘
```

- **枠は全ジャンル同一**、テンプレで**バーの中身（主/副フック）だけ差し替え**。指が位置を覚える＝負担減。
- 自動フィルの格納先は spec-A1：Eventの同一性属性（タイトル/原題/シリーズ/種別/URL）とVisitの回属性（鑑賞日/評価…）へ振り分け。取得メタの表示専用値は unitFieldsRaw、クエリする値（成果/評価/巻数/出演者）は個別プロパティ/人物マスターへ。
- **自動フィルは"生きた参照＋スナップショット"**：場所/人物/酒蔵はマスター参照＋記録時スナップショット（A1 §6・消えても記録は無傷）。

### ジャンル別プレースホルダ（マイクロテキスト）
| ジャンル | プレースホルダ |
|---|---|
| 映画 | タイトルで探す（例：ひつじ探偵団） |
| 書籍 | タイトルで探す、または裏のISBN（978〜）を撮る |
| おでかけ施設 | 施設を選ぶ（テーマパーク・水族館・動物園） |
| 御朱印 | 参拝した寺社を地図・名前・現在地から選ぶ |
| 酒 | ラベルを撮る、または銘柄で探す（さけのわ2500超・自分の記録・酒蔵） |
| 観劇 | 公演のURLを貼る（公演名・ビジュアルを取得）、または会場を選ぶ |
| 美術展 | 展覧会のURLを貼る、または会場を選ぶ |
| ライブ | ツアー/公演のURLを貼る、または会場を選ぶ |

## 4. ガイダンス層（迷わせない仕掛け）★2026-07-06 ユーザー要望

「フックが賢くても"何をすれば"が見えないと迷子になる」。**点で出して・学んだら消える**を原則に、美しさを損なわず誘導する。

1. **初回起動オンボーディング**：美しい1〜2画面（コア体験の一文＋「＋で記録する」だけ）。**スキップ可・長いチュートリアルにしない**。
2. **テンプレ初回コーチマーク**：そのジャンルを初めて開いた時だけ主フックを1行で指す（例：映画「タイトルを打つと候補が出ます」）。**数回で消え二度と邪魔しない**。
3. **スマートバーのプレースホルダ（§3表）**＝point-of-needのマイクロコピー。例付きで"何をすれば"を明示。
4. **空状態ガイド**：記録ゼロのカテゴリは「最初の記録をしよう」＋大きな＋。
5. **プレビューの安心コピー**：「自動で入りました。違ったら直せます」（誤取得で泣かせない）。
6. **必須の理由を添える**：御朱印「場所を選ぶと保存できます（地図に残ります）」＝なぜ必須かをその場で。
7. **段階的開示（progressive disclosure）**：主フック→自動フィル→自分の分、の順。**一度に全項目を見せない**。
8. **副フックの発見性**：主で見つからない時の逃げ道（URL/手動）をラベル付きで常設・でも控えめ。
9. **マイクロコピー原則＝生活語、ただし誤操作を防ぐ語は明示★2026-07-06 ユーザー指摘**：既定は生活語で誘導（「タイトルで探す」等・「ISBN」を不用意に多用しない）。**ただし、正確な語が具体的な失敗を防ぐ場合は明示する**——書籍のバーコードは日本の本に**2本並ぶ**（上＝ISBN「978〜」／下＝価格・分類コード「192〜」）ため、下を撮ると引けない。よって**「ISBN（978〜）を撮る」と明示**し、スキャナは**978/979始まりのみ受理・192始まりは「価格のバーコードです。上のISBNを読み取ってください」と誘導**（自動リジェクトで失敗させない）。ここは"使用者が学習する"前提でよい。番号直接入力は"番号で入力"として副に置く。各ジャンルも同原則（生活語＋誤操作防止語だけ明示）。

## 5. 負担を減らす原則（まとめ）

1. **1画面・1バー・同じ位置**：ジャンルが変わっても入り口の形は不変。
2. **候補は視覚で選ぶ**（タイトル＋年＋サムネ／施設名＋地図ピン）＝打鍵を最小化。
3. **自動フィルは勝手に確定しない**：必ずプレビュー→直せる。
4. **必須は極小**：日付は既定=今日、位置必須は御朱印だけ。他は空でも保存できる。
5. **自分の分（評価・感想）は後回しOK**：メタだけで保存→後で足せる（存在＝入力済み）。
6. **どのフックが無くても手動で必ず通る**（失敗しない設計）。

---

## 6. 持ち越し・要確定

- 各外部フックの実装可否・スコープ（TMDb/openBD/**NDL Search**はv1、URL-OGP/OCRの精度チューニング範囲）。書籍の奥付OCRの要否（同人/古書を撮るか）はv1で調整。
- 主フックが弱い観劇/美術展/ライブの"入り口強化"余地（チラシ写真→アイキャッチ、チケサイトOGPの活用範囲）。
- add-next（書籍の続巻・御酒印→酒）と統一バーの関係（既存記録から派生する入り口の置き方）。
- オンボーディング/コーチマークの具体UI・出現回数はデザインフェーズで確定。
