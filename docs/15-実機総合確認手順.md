# Favoreco 実機総合確認手順

> 対象: v1実装の総合確認
> 更新日: 2026-07-12
> 前提: 実データを入れる前に、まずテスト端末または削除可能な状態で行う

## 0. 準備

- Xcodeで `favorecoAPP.xcodeproj` を開く
- Schemeは `favorecoAPP`、実行先は確認するiPhoneを選ぶ
- Signing & CapabilitiesでTeamが `GDXKVW7W5X`、Bundle IDが `com.nori.favoreco` であることを確認
- iPhoneがApple Accountへサインインし、iCloud DriveがONであることを確認
- 通知、写真、カメラ、カレンダーの許可ダイアログは、最初は許可して確認する
- XcodeのSchemeには `Favoreco.storekit` が設定済み。ローカル課金確認ではそのまま使う
- 最低対応はiOS 18.0。まず手元のiOS 26実機で本手順を進め、iOS 18はSimulatorと公開前TestFlightで同じ主要フローを再確認する

iOS 18 Simulatorが一覧にない場合:

1. Xcode > Settings > Componentsを開く
2. iOS 18 Simulator runtimeをダウンロードする
3. Window > Devices and Simulators > SimulatorsでiOS 18のiPhoneを追加する
4. 最低限、初回起動、仮データ、記録保存/編集、Map検索、4タブ、設定を確認する

不具合記録には、画面名、操作手順、期待結果、実際の結果、スクリーンショット、発生時刻を残す。

## 1. 初回起動

1. iPhoneからFavorecoを削除する
2. XcodeからRunする
3. 初回ジャンル選択を開く
4. すべてのチェックを外そうとする
5. 映画、観劇、本など複数を選び完了する

期待結果:

- クラッシュせず起動する
- 0ジャンルでは完了できず、必ず1ジャンル以上が残る
- 完了後にHomeが表示される
- アプリアイコン下の表示名が `Favoreco`
- Home / 記録 / 追加 / カレンダー / 統計の5項目が同じ高さ・同じ扱いで表示される
- 追加だけが円形に浮いたり、タブバーから持ち上がったりしない

## 2. 仮データ

1. 右上プロフィールから設定を開く
2. 開発 > `写真付き仮データを追加` を押す
3. Home、記録、カレンダー、統計を順に見る
4. 各ジャンルを切り替える
5. 設定 > 開発 > `仮データを削除` を押す

期待結果:

- 有効な各ジャンルに1件以上の仮記録が入る
- 写真がHomeギャラリー、一覧、詳細に表示される
- 写真が真っ白、極端な縦横つぶれ、別記録との取り違えにならない
- 削除後は仮データだけが消え、手入力データは消えない

## 2.1 ローカルストア復旧画面のDEBUG診断

1. 既存の記録件数を控える
2. 設定 > 開発者メニュー > 診断で `次回起動で復旧画面を診断` を押す
3. アプリを終了し、Xcodeからもう一度Runする
4. `DEBUG診断中` と `記録データを開けませんでした` が表示されることを確認する
5. `診断を終了` を押し、案内どおりアプリを終了する
6. Xcodeからもう一度Runし、通常画面へ戻ることを確認する
7. 記録件数が手順1と一致することを確認する

期待結果:

- 診断中はHome、記録追加、設定など通常UIへ移動できない
- 実ストア、記録、写真、実際の起動エラー値を変更しない
- 診断中は標準ジャンルseed、通知移行、自動バックアップを実行しない
- 診断終了後の再起動で通常画面へ戻り、既存データが維持される
- Releaseビルドには診断入口を表示しない

## 3. 記録追加・編集

1. 下部の `追加` から任意ジャンルへ記録を追加
2. 基本情報、人物・団体、写真、OCR、金額、公式情報、メモ、詳細オプションを開閉
3. 写真を10枚追加し、カバー写真と表示比率を選ぶ
4. 公式URLを保存し、タイトル候補を取得
5. 住所つき会場を選択または入力
6. 保存する
7. 保存後の詳細画面から編集し、タイトル、日付、会場、写真を変更

期待結果:

- アコーディオンを閉じても入力値が消えない
- 保存後は編集画面ではなく記録詳細を開く
- 写真は長辺1600px、選択した65%/85%品質で保存される
- カバー写真と比率がHome、一覧、詳細で一致する
- 無料状態では11枚目を追加できず、既存10枚は消えない
- URL候補は利用者が選ぶまで自動上書きしない

## 4. Map・場所マスター

1. 同名施設が別地域にもある会場名を入力する
2. 正しい住所を登録する
3. 保存後に詳細画面の地図を開く
4. 同じ会場を別記録で候補から選ぶ
5. 設定の場所マスターを開き、全国／地方／都道府県を順に切り替える
6. 都道府県未設定の旧データがある場合は`未設定`で絞り込み、編集画面で都道府県を設定する

期待結果:

- 施設名は表示から消えない
- 地図位置は施設名検索ではなく登録住所を優先する
- 場所マスター候補から名称、住所、座標がまとめて再利用される
- 重複候補がある場合、利用者が統合するか判断できる
- 地方選択時は所属する都道府県だけを選択でき、県選択と検索文字列の両方が反映される
- 都道府県未設定では場所マスター編集を保存できず、住所に都道府県を含めるとピッカーへ自動反映される
- 0件、1件、大量件数、未設定データでも表示件数と空状態が崩れない

## 5. 人物・団体マスター

1. 人物・団体を複数登録し、役割を設定する
2. 別記録で同名候補を入力する
3. 既存候補を選択する
4. 重複候補の統合を試す

期待結果:

- 人物本体と出演/監督/著者などの役割リンクが分かれて保存される
- 同名候補が提示されても自動統合されない
- 統合後も既存記録とのリンクが失われない

## 6. 予定・チケット

1. 下部の `追加` > `予定・チケットを追加`
2. `気になる`、`抽選応募予定`、`発売待ち` を作る
3. 抽選ありの申込を作り、当落待ち > 当選 > 入金待ち > 発券待ち > 発券済み > 参加済みへ更新
4. 落選の申込も1件作る
5. プレイガイドを選択し、購入URLを確認
6. プレイガイド未指定の予定で公式URLを開く

期待結果:

- 状態に応じた日付、金額、座席、名義、プレイガイドが保持される
- プレイガイド指定時はDBのURLをワンタップ入力できる
- 個別購入URLがあればそれを優先し、なければプレイガイド、さらに未指定なら公式URLを開く
- 当選率は当選系と落選が確定した抽選だけを分母にする
- アーカイブした予定/申込は通常一覧と統計から除外される

## 7. 通知

1. 設定 > 通知でマスターをON
2. 公演前日/当日が初期ONであることを確認
3. 申込締切、当落、入金、発券、FC期限を必要なものだけON
4. 開発 > 通知診断から短時間のテスト通知を予約
5. アプリを前面、バックグラウンド、終了状態で確認
6. 月刊/年間通知のテスト遷移を確認

期待結果:

- 利用者がONにした通知だけが予約される
- 前面でもバナー、音、バッジが表示される
- 通知タップで対応画面を開く
- 月刊は前月、年間は前年のレポートを開く
- 権利がない場合、Premium向けレポート通知は予約されない

## 8. カレンダー

1. 予定詳細から `カレンダーに追加`
2. iOSカレンダーを選び保存
3. Google Calendarアプリが入っている場合は共有先/対象カレンダーとして選べるか確認
4. 設定 > 表示で外部カレンダー重ね表示をON
5. Favoreco側の予定を変更し、片方向更新を確認

期待結果:

- 初回だけ日本語の権限説明が表示される
- 手動追加は無料で使える
- 読み取り許可後、外部予定がFavorecoカレンダーへ重なる
- Favorecoが作成した予定だけを更新対象にし、外部予定を勝手に変更しない
- Google CalendarをiOSのカレンダーアカウントへ登録している場合、EventKit経由で表示・追加できる

## 9. 統計・レポート

1. 無料状態で統計を開く
2. DEBUG権利切替またはStoreKitローカル購入でPro以上にする
   - 設定 > 開発の権利表示で、写真上限、自作ジャンル、同期が選択プランと一致することを確認する
3. 直近12か月グラフ、ジャンル構成、予定・チケット統計を見る
4. 金額の目アイコンを押す
5. 月刊/年間Favorecoを開き、過去期間へ移動
6. 画像を生成して共有シートを開く

期待結果:

- 無料ではロック対象の数値をぼかさず、機能説明と必要プランを表示する
- 0件の月も12か月グラフから消えない
- ジャンルが6種類以上では上位5件とその他へまとまる
- 金額は初期状態で見えず、目アイコン操作後だけ見える
- 共有画像と共有文に金額を含めない

## 10. バックアップ・復元

1. JSONとCSVを書き出す
2. 写真付き完全バックアップをFilesへ書き出す
3. 自動バックアップを手動実行する
4. JSONをUUIDマージ復元する
5. 完全バックアップを復元する
6. 同じバックアップをもう一度復元する

期待結果:

- JSON/CSVは写真本体を含まない
- 完全バックアップは写真を含む
- 2回目の復元で同じUUIDの記録が無制限に重複しない
- 人物、場所、予定、チケット、写真の関係が復元される
- 空き容量不足時は作成前に止まり、既存データを壊さない

## 11. iCloud同期

可能なら同じApple AccountのiPhone 2台、またはiPhoneとiPadで行う。

1. 端末AでPremium権利を有効化
2. 設定 > 同期・バックアップでiCloud同期をON
3. アプリを終了して再起動
4. `現在の保存先: 端末 + iCloud` と `起動時の同期接続: 接続済み` を確認
5. 端末Aで記録、写真、予定、人物、場所を追加
6. 端末Bで同期ONにして起動し、Wi-Fiで待つ
7. 端末Bで編集し、端末Aへ反映されるか確認

期待結果:

- ON/OFF変更は説明どおり次回起動から反映される
- 初期化失敗時はローカルへフォールバックし、既存データを削除しない
- 写真を含む関係データが同期される
- 同じ項目が無限複製されない
- 初回大量同期中もHomeと一覧が操作不能にならない

## 12. StoreKit

ローカル確認:

1. Xcode > Product > Scheme > Edit Scheme > Run > Optionsで `Favoreco.storekit` を確認
2. 無料版、Pro、Premium月額、Premium年額、Premium永久アップグレード、Premium永久版を順に確認
3. Transaction Managerで購入を削除・期限切れにして権利変化を確認
4. `購入を復元` を確認

権利の組み合わせ確認:

1. Transaction Managerで全購入を削除し、無料へ戻る
2. Premium永久アップグレードだけを購入し、Premium永久権利にならないことを確認
3. 続けてProを購入し、Premium永久権利になることを確認
4. 購入をすべて削除し、Premium永久版を直接購入して同じ権利になることを確認

Sandbox/TestFlight確認:

- StoreKit Configurationを使わないArchive/TestFlightで行う
- Sandbox Apple AccountまたはTestFlight購入を利用する
- 商品名、価格、期間、購入、キャンセル、復元、失効を確認する

期待結果:

- Proはローカル拡張機能と写真無制限を永久利用可能。自作ジャンルと同期は含まない
- Premium月額/年額はPro機能、自作ジャンル、写真無制限、同期を契約中のみ利用可能
- Premium永久アップグレードはProとの両方を所有した時だけPremium機能を永久利用可能
- Premium永久版は単体でPro機能、自作ジャンル、写真無制限、同期を永久利用可能
- 失効しても保存済みデータを削除せず、新しい有料操作だけを止める

## 13. 写真負荷

1. 1記録へ20枚を追加できる権利状態にする
2. 85%で20枚保存し、Home、一覧、詳細を往復
3. 20枚の記録を複数作り、スクロールと起動時間を見る
4. 設定 > データ管理で写真容量と10,000枚時の推定値を見る
5. バックアップとiCloud初期同期中の発熱、メモリ、操作感を確認
6. Pro以上で1記録50枚まで追加し、容量付きの注意表示後も追加できることを確認
7. 無料版で保存済み写真を削除予定にし、空いた枠へ同じ編集画面内で写真を追加する

期待結果:

- 一覧は原寸画像ではなくサムネイルを読み、スクロールが停止しない
- 画面表示だけで全PhotoBlobのDataを一括ロードしない
- 保存中にアプリが落ちず、失敗時に壊れたPhotoBlobを残さない
- 初回10,000枚相当の同期は時間がかかる前提だが、記録本体を失わない
- 50枚以上は処理時間の注意だけを表示し、Pro以上の追加を止めない
- 削除予定の保存済み写真は即座にグリッドと件数から外れ、無料枠が戻る

## 14. 削除・アーカイブ

1. 記録、対象、予定をアーカイブして復元
2. キャッシュ削除、写真キャッシュ削除を実行
3. 完全バックアップを作成
4. `全データ削除` の確認文字を入力し実行

期待結果:

- アーカイブは削除ではなく、通常一覧から隠すだけ
- キャッシュ削除で原本記録・写真を消さない
- 全データ削除は二段階確認なしでは実行できない
- 削除後は標準ジャンルを再生成し、初回ジャンル選択へ戻る

## 合格条件

- クラッシュ、データ消失、別記録への写真混入がない
- 初回起動、保存、編集、削除、復元、再起動が一巡する
- 権限拒否でもアプリ全体は使用でき、対象機能だけが止まる
- 無料版/Pro/Premium/Premium永久版の境界が仕様表どおり
- 同期OFFでは完全ローカル、同期ONでは再起動後にCloudKitへ接続する
- 主要画面で文字切れ、重なり、タップ不能がない
