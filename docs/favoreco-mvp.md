# favoreco MVP定義

> 最終更新: 2026-07-04

## v1 に入れる（コア体験「記録が美しい思い出になる」だけで動く最小）

1. **RecordCategory**: テンプレート7種のプリセット（観劇/美術展/ライブ参戦/映画/酒/テーマパーク/書籍）＋自作カテゴリ作成・編集
2. **記録**: Event＋Visit（写真複数・日付・会場・テンプレ別フィールド・文字数無制限の感想・タグ・同行者・評価軸/成果）
3. **横断コレクション項目**: 半券・印・カード・スタンプの写真＋メモ
4. **ホーム**: カテゴリタブ＋タイル閲覧（Mystorium型：タップ展開→編集分離）
5. **統計**: カテゴリ別＋**カテゴリ横断**の基本セット（月次バー・BEST・レーダー・タグ頻度）
6. **シェアカード**: 基本デザイン（記録1件カード＋ランキング）
7. **検索・カレンダー**: Mystorium資産流用
8. **バックアップ**: zipエクスポート/インポート（`DataExporter` 流用。「データを人質にしない」の実装）
9. **買い切りIAP**: 無料制限（設計は別途）＋買い切りアンロック

## v1 に入れない（v2以降）

1. **サブスク機能すべて**: CloudKit同期 / 自動バックアップ＋世代復元 / 思い出の再提示（opt-in・ランダム表示） / 月次リキャップ / 限定デザイン定期追加 —— サブスク自体をv2で導入。**ただし同期のデフォルト構造（docs/07）に合わせ、v1スキーマ・写真保存形式はCloudKit互換で作る**
2. **年間まとめ**: 買い切り側の目玉だが v1.x（初年度の年末まで）に後追い。v1の統計基本セットを土台にする
3. 地図表示（MapKit画面）・位置情報起点の機能
4. 通知全般（チケット販売通知含む）
5. 追加テンプレート（食べ歩き・温泉・スポーツ観戦・御船印等）——自作カテゴリで代替可能なため
6. 半券OCR・スキャン整理
7. iPad最適化・Mac版・英語対応
8. ソーシャル機能・作品DB連携（コンセプトの「やらないこと」＝恒久除外）

## v1 の成功条件（検証可能な形で）

1. テンプレ6種＋自作カテゴリのすべてで「記録作成 → 一覧 → 詳細 → 統計 → シェアカード」の動線がクラッシュ・データ欠損なく回る
2. 空データ・0件・1件・大量（1,000件）の各パターンで表示破綻がない
3. zipエクスポート → 全削除 → インポートで写真含め完全復元できる
4. 開発者自身の過去の体験30件以上を実際に移行し、**柱2ジャンル以上で週1回以上開きたくなる**（コア体験の主観検証。ならなければ何が足りないかをlogに残して改修）
5. カテゴリの追加・リネーム・削除（記録は消えない）が既存記録を壊さない

## ロードマップ（粗）

- **v1**: 上記（買い切りの中核のみ・ローカル完結）
- **v1.x**: 年間まとめ（年末商戦＝Xバイラルの山に合わせる）・追加テンプレ数種
- **v2**: サブスク導入（CloudKit同期・自動バックアップ・思い出再提示・月次リキャップ・限定デザイン）
- **将来**: 半券OCR・共有アーカイブ・Mac
