# ②-A / spec-A4 テンプレ別プリセット具体リスト

> ステータス: 清書（2026-07-06）
> 位置づけ: ②-A のプリセット分冊。各テンプレの**評価軸名・成果選択肢・形態選択肢・作品ジャンル選択肢・スペックチップ・種別サブレイヤーの中身**を具体化。フィールドの器はA3、種別の仕組みはA2。
> 凡例: **【確定】**＝08のヒアリングで確定済み ／ **【提案】**＝08に明記が無く本書で埋めた案（要ユーザー確認）。すべて**ユーザー編集可**（プリセットは初期値）。
> 関連: A2（種別サブレイヤー）／ A3（フィールド）／ 08 §4-2〜§4-11

---

## 0. 全テンプレ共通の考え方

- プリセットは**初期値**。ユーザーは選択肢の追加/リネーム/削除ができる（OFFにしても入力済みデータは消さない・08 §4）。
- 選択肢マスター（成果/形態/ジャンル/評価軸）は**カテゴリ（＋種別）スコープ**で持つ（A2解決）。
- 成果・評価軸は「同じユニットを使うカテゴリで同じ統計が成立する」ためキーで管理。

## 1. 総合評価（RESONANCE型）

- 全カテゴリ共通の**単一総合評価**（Visit・A3土台）。Mystorium由来の呼称「RESONANCE」を踏襲。
- **【確定 2026-07-06】★0.5刻みの5段階（0.5〜5.0）**。Double(0.5刻み)で保存し統計・ソートに使う（Filmarksと同じ半刻み）。
- U7評価軸レーダーは総合評価とは別（軸別の詳細・オフ開始のテンプレが多い）。

### ★成果(U8)は全ジャンル廃止（2026-07-06 ユーザー判断）
「総合評価があれば十分・争うものでもない」。**U8成果を全テンプレから外す**（08マトリクスの●も無効）。ただし**書籍の「読了/積読/中断/再読」は"評価"でなく"読書状態"**なので、成果とは別の**状態フィールド**として残す（§9・要ユーザー確認）。

## 6. 酒【確定＋日本酒プリセット】

### 酒種別サブレイヤー（A2）
- 種別：**日本酒 / ビール / ウィスキー / ワイン / 焼酎 / その他**（v1は下地のみ通し、中身は**日本酒プリセットだけ実装**）。

### 日本酒プリセット
- **味覚マップ2軸(U17)●**：横=甘辛度（辛口⇔甘口）／縦=濃淡度（淡麗⇔芳醇）。4タイプ表示＝**薫酒 / 爽酒 / 醇酒 / 熟酒**。さけのわ数値を参考デフォルト表示。
- **評価軸レーダー(U7)○（さけのわ6軸準拠・確定）**：華やか / 芳醇 / 重厚 / 穏やか / ドライ / 軽快（対の関係：華やか⇔穏やか・軽快⇔重厚）
- **官能メモ(U10)○**：香り → 味わい → 余韻
- **スペック表(U9)●チップ**：蔵元 / 銘柄名 / 使用米 / 酵母（＋自由追加）
- **昇格数値列（Event個別・検索/グラフ）**：精米歩合 / 日本酒度 / 酸度 / アルコール分
- ~~成果(U8)~~ 廃止（§1）。「リピート買いたい」はタグ or リピート機能で代替

## 10. カスタムカテゴリ

- 上記は「保存済みプリセット」。新規カスタムは 名前・アイコン・色 → ユニット選択 → 各選択肢（成果/形態/ジャンル/評価軸）を編集 → 完成（08 §4）。
- 味覚2軸(U17)の軸ラベルもカテゴリ別編集可（コーヒー=酸味×コク 等に転用）。

## 11. 持ち越し（A5/A6へ）

- **解決済み（2026-07-06）**：RESONANCE=★0.5刻み5段階／成果=全廃／さけのわ6軸=確定／音楽・書籍ジャンル=調査反映／映画評価軸=脚本/演出/演技/映像/音楽（既定オフ）／美術展=総合評価のみ推奨。
- **残る要確認**：書籍の「読書状態(読了/積読/中断/再読)」を残すか（成果でなく"状態"として保持の提案）。
- A5：選択肢を使った統計（ジャンル別・軸平均・味覚マップ集計・**読書状態別＝積読リスト**等）。
- A6：選択肢マスターのkey/表示名分離（A2同様・リネームで既存記録を壊さない）。
