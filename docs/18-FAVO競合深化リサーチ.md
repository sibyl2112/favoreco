# FAVO競合深化リサーチ

> 状態: 方向Bを採用・段階実装中
> 調査日: 2026-07-21
> 対象: ユーザー提示の参考アプリ18本、Favoreco現行FAVO第1段階
> 注意: 評価件数・料金・機能は変動する。口コミ欄は全件の統計分析ではなく、App Storeで確認できた代表的な要望・不満の定性整理として扱う。

## 1. 今回の問い

1. `FAVO`を「推しを使わない人」にも自分ごととして感じられる場所にできるか
2. 競合の専用機能を足し合わせず、Favorecoの既存データからより強い体験を作れるか
3. Home、FAVO、ジャンルトップ、カレンダー、統計の役割を重複させずに改善できるか

## 2. 先に結論

現行FAVOの最大の課題は、画面名と保存構造のずれにある。

- 画面名と説明は、人物・場所・作品・体験を含む「好き全般」を表している
- しかし利用者が明示的にFAVOへ登録できるのは`FavoriteProfile`を持つ人物・団体だけ
- 場所・作品・体験は検索入口として並ぶだけで、「自分のFAVOとして集める」「順番を決める」「その対象の物語を見る」ができない
- 推し0件時の最近の思い出は全記録共通であり、その人が何を好きかを表す画面にはまだなっていない

したがって、人物用の`FavoriteProfile`を無理に汎用化するのではなく、人物・作品・場所・タグ等を同じFAVO面へ固定できる軽量な横断モデルを別に置く方向が最も自然である。

```text
FavoriteProfile
  人物・団体だけが持つ感情的プロフィール
  推し始めた日 / 呼び名 / 推しカラー / 記念日 / 個人メモ

FavoPin
  FAVOトップへ置く横断ブックマーク
  person / event / place
  表示順 / 短い自分用タイトル（入力UIは後続）
```

FAVOは、`好きな対象を固定する`ことと、`既存記録からコレクションが自動で育つ`ことを組み合わせる。記録自体は複製しない。

## 4. 口コミから横断して見えた成功条件

### 4-1. 高評価につながる共通点

1. 入力の最初が軽い
   - 写真、URL、バーコード、3項目だけで開始できる
2. 保存直後に成果物ができる
   - 半券、棚、ポスター面、年鑑、カウントダウン等へ変わる
3. 次の行動が1つに絞られている
   - 次のライブ、次の締切、あとでやる、Wishlist
4. たまった記録が自己理解へ変わる
   - よく行く場所、初めて、回数、費用、好み、年別ハイライト
5. 対象への愛着が画面に出る
   - Favorite、推し色、写真、固定、呼び名

### 4-2. 不満になりやすい共通点

1. 保存・同期・写真選択が信用できない
2. 同じツアー、複数公演、複数日を毎回手入力する
3. カレンダーへ情報を詰め込みすぎて読めない
4. 好きな対象を上へ固定できず、統計や一覧に埋もれる
5. 並び順・表示モードが勝手に戻る
6. 外部DBの未収録・誤記を利用者が直せない
7. 数字や他人の評価を見たくない時にも強制表示される
8. 高度統計、広告非表示、十分な記録枚数が継続課金の壁になる

## 5. Favoreco現行FAVOの監査

### 5-1. すでに競合より強い土台

- 1件のVisitを複製せず、人物・作品・場所・タグから横断して見られる
- EventリンクとVisitリンクの重複を`FavoSnapshot`で除外できる
- 推し解除や人物統合で記録を失わない
- Homeに期限、カレンダーに日程、統計に数値、FAVOに記憶という役割分担がある
- OCR、URL候補、写真、チケット工程、人物・場所マスター、バックアップを既に持つ
- 非公開・ローカル/CloudKit・エクスポート可能という、SNS疲れとデータ人質への対抗軸がある

### 5-2. 現状の不足

| 不足 | 現在 | 目指す状態 |
|---|---|---|
| FAVO対象 | 人物だけが明示登録可能 | 人物・作品・場所・タグ/コレクションを固定可能 |
| 推し0件の価値 | 全体の最近記録と管理画面への入口 | 自分で選んだFAVOと自動コレクションが主役 |
| 見返す成果物 | リスト行が中心 | カバー付きFAVOカード、半券/写真の棚、物語タイムライン |
| 自動発見 | 回数と最近記録 | 初回、最新、今年、よく行く場所、初めて、再訪、On This Day |
| 反復入力 | 記録ごとに入力 | OCR/URLに加えて前回・同ツアーから複製して差分編集 |
| 信頼性の見せ方 | iCloud内部挙動に依存 | 保存済み・同期中・未同期を必要な場所だけ説明可能 |

## 6. FAVOトップの方向案

### A. 人物中心を維持する最小改修

- `FavoriteProfile`だけを使う
- 推し0件時は最近の人物・場所・作品を自動表示
- 文言を`推し`から`FAVO`へ中立化

利点は実装が小さいこと。欠点は、場所や作品を利用者がFAVOとして選べず、名称と構造のずれが残ること。

### B. 横断FAVO＋自動コレクション（推奨）

- `FavoriteProfile`は人物固有の感情情報として残す
- `FavoPin`で人物・作品・場所をFAVOトップへ固定（タグ/スマートコレクションの固定は後続）
- 固定対象ごとに既存のPlan/Visit/Photo/TicketAttemptを集計し、記録の複製はしない
- 固定していない人にも、記録から`最近よく行く場所`、`今年増えた作品`、`写真のある思い出`等の自動コレクションを提案

FAVOの意味が推しの有無に左右されず、既存の人物リンク・場所マスター・Event/Visit分離を最も活かせる。

### C. 完全自動のMY ARCHIVE

- 利用者によるFAVO設定をなくす
- 記録頻度や最近性だけで人物・作品・場所の棚を自動生成

入力は最小だが、偶然回数が多いものと本当に好きなものを区別できず、愛着や自分で選ぶ感覚が弱い。

## 7. 推奨する画面構成

方向Bを採用する場合の第1段階。

```text
FAVO
├─ MY FAVO
│  └─ 人物 / 作品 / 場所 / コレクションを最大4件
├─ STORIES
│  └─ 初めて / 最新 / On This Day / 再訪
├─ コレクション
│  └─ 写真 / 半券 / 場所 / 今年 / ジャンル横断の自動まとめ
└─ EXPLORE
   └─ 人物・団体 / 場所 / タグ・同行者 / 作品・体験 / 全記録検索
```

### 表示原則

- MY FAVO 0件でも空の設定画面にしない。コレクションと最近の物語を先に見せる
- 推し登録者は人物カードで`推して○日`を表示できるが、作品や場所には押しつけない
- Homeは期限・対応事項・直近予定、FAVOは好きな対象と記憶を担う。FAVOトップへ予定一覧を置かず、対象詳細だけに次の予定1件を文脈補助として置く
- 統計はFAVOトップで大量に並べず、対象詳細または統計タブへ送る
- 自動コレクションの並びは安定させ、利用者が固定・非表示にできる
- FAVO削除はピンを外すだけで、元記録・人物・場所・作品を削除しない

## 8. 実装優先度案

### P0: FAVOの意味を成立させる

- `[実装済み]` `FavoPin`を人物専用`FavoriteProfile`と分離し、人物・作品・場所の所属と表示順だけを保存
- `[実装済み]` 人物・作品・場所を最大4件混在で追加／解除／並べ替え
- `[実装済み]` MY FAVOカードと対象別詳細へ遷移し、次の予定1件、記録全件、写真数、記録済み支出を集計表示
- `[実装済み]` 推し0件・記録0件でも作品／場所の追加、STORIES、コレクション、探すを利用可能
- `[実装済み]` JSON／完全／自動バックアップ、全データ削除、人物／場所統合、CloudKitスキーマへ接続
- `[実機待ち]` 再起動後の順序、既存ストア移行、バックアップ往復、ライト／ダーク、Dynamic Typeを実機確認

### P1: 見返したくなる成果物

- `[実装済み]` STORIESの初回/最新/On This Day
- `[部分実装]` 写真・場所・今年・ジャンルの自動コレクションと、条件別Visit一覧への遷移。半券の専用コレクションは未実装
- 記録からFAVO候補を提案し、利用者が確定
- 同ツアー/前回記録からの複製と差分入力

### P2: 愛着と外部出力

- 年間FAVO、共有カード、ウィジェット
- `[部分実装]` 支出は人物・作品・場所ごとの年別／ジャンル別内訳まで実装。写真ギャラリーと会場地図は未実装
- 保存・同期状態の診断表示

## 9. 今回は追わないもの

- Goodreads、Filmarks、Letterboxd型の公開レビューSNS
- MysteryLog型のチャット、フォロー、公開成功率
- シアティ型のポイント・スタンプ・ゲーム
- 全会場の座席図を自前保有すること
- 公演/作品/銘柄DBの網羅性競争
- FAVO内に汎用タスク管理や共有カレンダーを再実装すること

## 10. 完成条件のたたき台

方向BのP0は、コードと保存経路では1〜7を実装済みとし、8と実機での移行・再起動・復元確認を残す。

1. 推しを1人も登録しなくても、作品または場所をMY FAVOへ置ける
2. 人物・作品・場所を混在して最大4件固定し、順番が再起動後も保たれる
3. 1件の記録が人物・作品・場所それぞれのFAVOへ重複保存なしで現れる
4. FAVOを外しても元記録は消えない
5. 記録0件、FAVO0件、FAVOだけ、記録だけ、大量記録の各状態で行き止まりにならない
6. Homeの期限/予定、カレンダーの日程、統計の数値と役割が重複しない
7. JSON/完全/自動バックアップ、CloudKit、全データ削除で新モデルが欠落しない
8. iOS 18/26、ライト/ダーク、Dynamic Typeで固定カードと入口が読める
