# FAVO競合深化リサーチ

> 状態: 方向Bを採用・段階実装中
> 調査日: 2026-07-21（2026-07-22に推し活専用3アプリと市場資料を追補）
> 対象: ユーザー提示の参考アプリ18本、Favoreco現行FAVO第1段階
> 注意: 評価件数・料金・機能は変動する。口コミ欄は全件の統計分析ではなく、App Storeで確認できた代表的な要望・不満の定性整理として扱う。

## 1. 今回の問い

1. `FAVO`を「推しを使わない人」にも自分ごととして感じられる場所にできるか
2. 競合の専用機能を足し合わせず、Favorecoの既存データからより強い体験を作れるか
3. Home、FAVO、ジャンルトップ、カレンダー、統計の役割を重複させずに改善できるか

## 2. 先に結論

現行FAVOの最大の課題は、画面名と保存構造のずれにある。

- 画面名と説明は、人物・場所・作品・体験を含む「好き全般」を表している
- しかし利用者が明示的にFAVOへ登録できるのは`FavoriteProfile`を持つ人物・団体だけ
- 場所・作品・体験は検索入口として並ぶだけで、「自分のFAVOとして集める」「順番を決める」「その対象の物語を見る」ができない
- 推し0件時の最近の思い出は全記録共通であり、その人が何を好きかを表す画面にはまだなっていない

したがって、人物用の`FavoriteProfile`を無理に汎用化するのではなく、人物・作品・場所・タグ等を同じFAVO面へ固定できる軽量な横断モデルを別に置く方向が最も自然である。

```text
FavoriteProfile
  人物・団体だけが持つ感情的プロフィール
  推し始めた日 / 呼び名 / 推しカラー / 記念日 / 個人メモ

FavoPin
  FAVOトップへ置く横断ブックマーク
  person / event / place
  表示順 / 短い自分用タイトル（入力UIは後続）
```

FAVOは、`好きな対象を固定する`ことと、`既存記録からコレクションが自動で育つ`ことを組み合わせる。記録自体は複製しない。

## 4. 口コミから横断して見えた成功条件

### 4-1. 高評価につながる共通点

1. 入力の最初が軽い
   - 写真、URL、バーコード、3項目だけで開始できる
2. 保存直後に成果物ができる
   - 半券、棚、ポスター面、年鑑、カウントダウン等へ変わる
3. 次の行動が1つに絞られている
   - 次のライブ、次の締切、あとでやる、Wishlist
4. たまった記録が自己理解へ変わる
   - よく行く場所、初めて、回数、費用、好み、年別ハイライト
5. 対象への愛着が画面に出る
   - Favorite、推し色、写真、固定、呼び名

### 4-2. 不満になりやすい共通点

1. 保存・同期・写真選択が信用できない
2. 同じツアー、複数公演、複数日を毎回手入力する
3. カレンダーへ情報を詰め込みすぎて読めない
4. 好きな対象を上へ固定できず、統計や一覧に埋もれる
5. 並び順・表示モードが勝手に戻る
6. 外部DBの未収録・誤記を利用者が直せない
7. 数字や他人の評価を見たくない時にも強制表示される
8. 高度統計、広告非表示、十分な記録枚数が継続課金の壁になる

## 5. Favoreco現行FAVOの監査

### 5-1. すでに競合より強い土台

- 1件のVisitを複製せず、人物・作品・場所・タグから横断して見られる
- EventリンクとVisitリンクの重複を`FavoSnapshot`で除外できる
- 推し解除や人物統合で記録を失わない
- Homeに期限、カレンダーに日程、統計に数値、FAVOに記憶という役割分担がある
- OCR、URL候補、写真、チケット工程、人物・場所マスター、バックアップを既に持つ
- 非公開・ローカル/CloudKit・エクスポート可能という、SNS疲れとデータ人質への対抗軸がある

### 5-2. 現状の不足

| 不足 | 現在 | 目指す状態 |
|---|---|---|
| FAVO対象 | 人物だけが明示登録可能 | 人物・作品・場所・タグ/コレクションを固定可能 |
| 推し0件の価値 | 全体の最近記録と管理画面への入口 | 自分で選んだFAVOと自動コレクションが主役 |
| 見返す成果物 | リスト行が中心 | カバー付きFAVOカード、半券/写真の棚、物語タイムライン |
| 自動発見 | 回数と最近記録 | 初回、最新、今年、よく行く場所、初めて、再訪、On This Day |
| 反復入力 | 記録ごとに入力 | OCR/URLに加えて前回・同ツアーから複製して差分編集 |
| 信頼性の見せ方 | iCloud内部挙動に依存 | 保存済み・同期中・未同期を必要な場所だけ説明可能 |

## 6. FAVOトップの方向案

### A. 人物中心を維持する最小改修

- `FavoriteProfile`だけを使う
- 推し0件時は最近の人物・場所・作品を自動表示
- 文言を`推し`から`FAVO`へ中立化

利点は実装が小さいこと。欠点は、場所や作品を利用者がFAVOとして選べず、名称と構造のずれが残ること。

### B. 横断FAVO＋自動コレクション（推奨）

- `FavoriteProfile`は人物固有の感情情報として残す
- `FavoPin`で人物・作品・場所をFAVOトップへ固定（タグ/スマートコレクションの固定は後続）
- 固定対象ごとに既存のPlan/Visit/Photo/TicketAttemptを集計し、記録の複製はしない
- 固定していない人にも、記録から`最近よく行く場所`、`今年増えた作品`、`写真のある思い出`等の自動コレクションを提案

FAVOの意味が推しの有無に左右されず、既存の人物リンク・場所マスター・Event/Visit分離を最も活かせる。

### C. 完全自動のMY ARCHIVE

- 利用者によるFAVO設定をなくす
- 記録頻度や最近性だけで人物・作品・場所の棚を自動生成

入力は最小だが、偶然回数が多いものと本当に好きなものを区別できず、愛着や自分で選ぶ感覚が弱い。

## 7. 推奨する画面構成

方向Bを採用する場合の第1段階。

```text
FAVO
├─ MY FAVO
│  └─ 人物 / 作品 / 場所 / コレクションを最大4件
├─ STORIES
│  └─ 初めて / 最新 / On This Day / 再訪
├─ コレクション
│  └─ 写真 / 半券 / 場所 / 今年 / ジャンル横断の自動まとめ
└─ EXPLORE
   └─ 人物・団体 / 場所 / タグ・同行者 / 作品・体験 / 全記録検索
```

### 表示原則

- MY FAVO 0件でも空の設定画面にしない。コレクションと最近の物語を先に見せる
- 推し登録者は人物カードで`推して○日`を表示できるが、作品や場所には押しつけない
- Homeは期限・対応事項・直近予定、FAVOは好きな対象と記憶を担う。FAVOトップへ予定一覧を置かず、対象詳細だけに次の予定1件を文脈補助として置く
- 統計はFAVOトップで大量に並べず、対象詳細または統計タブへ送る
- 自動コレクションの並びは安定させ、利用者が固定・非表示にできる
- FAVO削除はピンを外すだけで、元記録・人物・場所・作品を削除しない

## 12. 2026-07-22追補: 推し活専用3アプリから見た勝ち筋

### 12-1. 結論

Favorecoは、もう1本の「推して何日＋かわいい着せ替え」アプリとして競争しない。

勝てる位置は、`推しを飾る入口`と`実際に行った・見た・使った体験の正本`を1本につなぐことである。FAVOで日数、写真、色、記念日を感情的に見せ、記録本体では公演・作品・場所・人物・チケット・写真・費用を構造化して残す。競合がプロフィール、日記、カレンダー、支出を別々の箱へ保存するのに対し、Favorecoは1件のVisitを人物・作品・場所・年・ジャンルから重複なく振り返れる点を中核価値とする。

最初に狙うのは、推しができたばかりの全ユーザーではない。ライブ、観劇、展覧会、遠征等の体験が増え、日数カウンターや自由日記では後から探せなくなった人を初期対象にする。

推奨ポジショニングは次のとおり。

> 推しを飾るだけじゃない。会った日、行った場所、使ったお金まで、好きの履歴がつながる。

### 12-3. レビューから確度が高い未充足需要

ユーザー提示画面とApp Storeで反復して確認できた要望を、重要度順に整理する。

1. **日付と通知を信用できること**
   - 初日を1日目にするか0日目にするかが、経過日数、次の節目、周年、通知のすべてで一致する
   - 通知本文だけで`誰の・何の記念日・いつ`か分かり、タップ後に該当FAVOへ進める
   - 独自の節目と通知時期を自分で選べる
2. **写真を切らず、増えても軽く見返せること**
   - 一覧は一定比率でも、詳細では写真全体を見られる
   - 1枚だけでも大きく見られ、複数枚はアルバムから元の記録へ戻れる
   - 写真50枚以上でもサムネイル生成とスクロールが破綻しない
3. **記録が入力した箱に閉じないこと**
   - 人物、作品、公演、場所、年、ジャンル、費用から同じ体験へ戻れる
   - カレンダー通知から対象、対象から当日の写真・チケット・感想へ連続して進める
4. **人に見られず、失わず、持ち出せること**
   - Face ID/パスコード、ローカル優先、広告トラッキングなし
   - バックアップ、機種変更、エクスポートの状態が利用者に分かる
5. **複数推しでも操作状態を失わないこと**
   - 最推しの固定を維持したまま、他のFAVOへすぐ移れる
   - 人物だけでなく、作品、グループ、場所、関係性もFAVOとして扱える

### 12-4. Favorecoが取るべき3層

| 層 | 約束 | 実装・商品判断 |
|---|---|---|
| 信頼 | 日付がずれない、通知が誰の何か分かる、保存した記録を失わない | 共通日付計算器、通知ID/本文の統一、同期診断、Face ID、JSON/完全バックアップを先行する |
| 接続 | 1回の記録が人物・作品・場所・予定・支出・写真へつながる | Visitを正本に保ち、FAVO側へ複製しない。通知や年間まとめも同じIDから生成する |
| 愛着 | 開くたびに好きが育ち、節目に見返したくなる | Hero、アイコン、複数記念日、ギャラリー、On This Day、年間FAVO、共有カード、ウィジェットを段階追加する |

優先順は必ず`信頼 → 接続 → 愛着`とする。装飾を先に増やしても、日付・通知・保存が信用されなければ長期記録アプリとして残らない。

### 12-5. 実装ロードマップの修正

#### P0: 信頼を商品化する

- 記念日計算を1つの純粋な計算器へ統合し、起点日、当日、翌日、100日、1周年、うるう日、タイムゾーン変更を自動テストする
- `初日を含める`設定を設ける場合は、日数、次回節目、残日数、通知、共有カードの全出力へ同じ値を渡す
- 基本通知は無料。通知本文を`FAVO名 / 記念日名 / 日付または残り日数`で統一し、該当FAVOへdeep linkする
- 同期中、最終同期、端末内のみ、要確認を必要な画面だけに表示する
- Face ID/パスコードと、利用者が実行できる書き出し・復元確認を用意する

#### P1: 競合にない接続を見せる

- FAVO詳細の同じタイムラインへ、記念日だけでなくVisit、Plan、TicketAttempt、写真、支出を統合表示する
- `通知 → FAVO → 対応する予定/記録 → 写真・チケット・感想`を2タップ以内で往復できるようにする
- 新規記録保存時に人物・作品・場所を候補として提示し、既存マスターへ接続する
- 前回公演・同ツアーから複製し、日付・会場・座席だけを差分編集できるようにする
- 写真／半券／会場の自動コレクションを、元Visitへ戻れる成果物として完成させる

#### P2: 見返しと課金価値を作る

- 年間FAVO、On This Day、節目カード、ウィジェット、共有カード
- Proは高度な表現と整理に課金する: 推しギャラリー15枚超、追加テーマ、独自節目、通知オフセット、詳細検索・統計、透かしなし共有カード
- Premiumは継続費用が発生する価値に限定する: CloudKit同期、自動バックアップ、定期的な年間/月間レポート、将来の外部候補補助
- 基本記録、基本日数、推し始めた日、誕生日、カスタム記念日1件、基本通知、推しギャラリー15枚、手動書き出しは無料に残す。複数のカスタム記念日、推しギャラリー16枚目以降はPro以上とする

### 12-6. 追わない機能

- ガチャ、ジェム、動画広告、連続ログインを中核ループにしない
- Oshibanaのウィジェット数、キーボード、壁紙、音楽・動画連携を初期段階で網羅しない
- 公開タイムライン、質問箱、チャット、フォローを作らない
- 写真日記の全文エディタ競争をしない。短い感想と写真を構造化された体験へ添える
- 外部の人物・作品画像を無断収集してマスターDB化しない
- 保存済みの個人記録を広告閲覧やガチャの報酬へ結び付けない

## 13. 人物登録・推し優先度・表現範囲の確定

### 13-1. 人物登録の3経路

人物の入口は次の3経路とし、内部ではすべて同じ`PersonMaster`を検索・再利用・作成する。

1. FAVOから推しを登録
2. 人物マスターから人物を登録
3. 公演情報のキャスト登録から人物を登録

人物マスター／キャスト登録では推し設定を必須にしない。推しまたは最推しにした時だけ`FavoriteProfile`を作り、MY FAVOへ表示する場合は`FavoPin`を接続する。FAVOからの登録でも先に既存人物候補を出し、同一人物の新規作成を避ける。

### 13-2. 最推しと通常の推し

- 最推しと通常の推しはどちらも複数登録可能
- 最推しを1件へ強制せず、UI上の代表だけを`メイン表示`1件として別に選ぶ
- メイン表示はFAVOの初期Heroと有料ウィジェットの既定対象にする
- メイン表示が外れた場合は、最推しの先頭、通常の推しの先頭の順で自動選択する
- 最推し同士、通常の推し同士は利用者が並べ替える
- MY FAVO最大4件は人物・作品・場所を混在表示する枠であり、人物の推し登録数上限にはしない

### 13-3. カスタマイズの境界

Favorecoは推し特化アプリではないため、全画面の着せ替え競争には入らない。

| 採用する | 採用しない |
|---|---|
| FAVO専用Hero写真 | アプリ全体の背景変更 |
| FAVOアイコン | 任意フォント変更 |
| FAVOカラー | 動くエフェクト、スタンプデコレーション |
| 呼び名、記念日、ギャラリー | 推しカラーによる本文全面の塗り替え |

FAVOカラーは専用ページのHero、アクセント、記念日、選択状態へ絞り、本文とナビゲーションはFavoreco共通の可読性を維持する。

### 13-4. ウィジェットの課金境界

- FAVOウィジェットはPro／Premium／Premium永久版限定
- 無料版はアプリ内のFAVO、日数、記念日、保存済み写真閲覧まで利用可能
- メイン表示を既定対象にし、最推し、通常の推し、または人物・作品・場所のMY FAVOへ切替可能
- プラン失効時もウィジェット設定、プロフィール、写真、記録は削除しない
- 初期版は日数、次の記念日、写真へ限定し、キーボード、壁紙、音声、動画連携は追わない

### 13-5. 複数記念日の課金境界

- 推し始めた日と人物マスターの誕生日は無料で、カスタム記念日の件数へ含めない
- 無料版はプロフィールごとにカスタム記念日1件まで追加・編集可能
- Pro／Premium／Premium永久版は複数追加、編集、並べ替え可能
- 無料へ戻った時に2件以上保存済みでも削除せず、FAVO詳細と管理一覧で全件閲覧できる
- 無料で2件以上保存済みの場合は全件を閲覧専用とし、Pro以上へ戻るまで追加・編集・並べ替えをロックする
- 基本通知は無料枠の記念日へ提供し、複数記念日の個別通知と通知オフセットはPro以上の後続実装とする
