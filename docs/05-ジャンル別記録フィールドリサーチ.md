# ジャンル別記録フィールドリサーチ（テンプレート設計の素材）

> 調査日: 2026-07-04
> 手法: ディープリサーチ（検索5系統・44エージェント・341ツール実行）。検証プロセスを通過した「検証済み」と、検索結果・要旨ベースの「検索ベース」を区別して記載。
> ※実行環境のネットワーク制限でソース本文の全文取得が大半403だったため、検証済みまで到達したのは観劇（シアティ）のみ。他ジャンルは検索ベース（出典URLは記載、実装前の②-A仕様化時に個別確認を推奨）。

---

## 総論：ジャンル横断で見えた5つの設計パターン

1. **「少数の構造化必須項目＋文字数無制限の自由記述＋複数写真」が実証済みの型**（検証済み・シアティ）。観劇ログは写真・タイトル・日付・開演時間・会場・座席・日記の7項目で、全部入力しなくても保存できる。
2. **記録は「本体＋周辺のモノ・装い」のセット**（検証済み・シアティ運営の用途提案）。チケット半券・アクスタ・当日コーデ・推しカラーネイル・観劇後の食事まで含めた「1日の思い出パッケージ」。
3. **半券写真＋メモはジャンル横断の共通仕様**。半券をTHEATER/LIVE/Cinemas/ART/TRAVELに分類してノートに貼る実践者が実在（＝favorecoのペルソナそのもの）。電子チケット化で「半券が残らない」問題があり、スクショ・画面写真の受け皿も必要。
4. **スペック値と官能評価は分離する**（酒・サウナ・温泉）。ウイスキーは「ラベル情報（銘柄・年数・度数・蒸溜所）」＋「香り→味わい→余韻の時系列3段階」。サ活は「温度・水風呂温度・混雑度」＋「感想」。Mystoriumの評価軸（AxisDefinition）構造がそのまま活きる。
5. **振り返りUIはカレンダー＋期間検索が中核**（検証済み・シアティ）。「あの舞台いつ観たっけ」という想起ニーズが設計の中心。サウナイキタイは統計レポート・1年まとめ（訪問都道府県・好みのタイプ）まで提供。

## ジャンル別フィールド表

凡例: ★=検証済み ○=検索ベース（出典あり）

### 1. 観劇・ミュージカル・2.5次元 ★検証済み

| 区分 | 項目 |
|---|---|
| 必須級 | 日付・タイトル・会場・**座席番号**・開演時間（マチネ/ソワレ） |
| あると嬉しい | キャスト（当日キャスト変更の記録）・同行者・観劇回数（リピート）・チケット代・先行/一般等の入手経路 |
| 写真・半券 | 半券・公演ポスター・アクスタ・当日コーデ・ネイル・観劇後の食事。複数枚必須 |
| 自由記述 | 文字数無制限の観劇日記 |

- 根拠: シアティの実装7項目（3-0票で検証済み）。座席記録を「エンタメファンのための」差別化点として公式が訴求
- 補強: 2.5次元ファンはLifebear等の汎用手帳で「先行・支払い・発券・感想」を代用中（専用の空白）○。Notion自作勢は公演名・日付・会場＋カレンダービュー構成○
- 注意: 「シアティが他ジャンルに流用されている」という主張は検証で否決（0-3）。横断需要の根拠には使わない

### 2. 美術館・博物館・展覧会 ○

| 区分 | 項目 |
|---|---|
| 必須級 | 展覧会名・会場・訪問日・会期 |
| あると嬉しい | お目当ての作品・音声ガイド有無・図録購入・混雑度・所要時間 |
| 写真・半券 | 半券＋**チラシ＋作品リスト**のセット保存文化がある。グッズ・館内撮影可作品 |

- 根拠: 展覧会半券管理コラム（ごじょ記）・チラシ整理実践記事。半券は展覧会名・会期・会場と紐付けて保管される

### 3. ライブ・フェス・推し活参戦 ○

| 区分 | 項目 |
|---|---|
| 必須級 | 公演名/ツアー名・アーティスト（推し）・日付・会場・座席/整理番号 |
| あると嬉しい | **セットリスト**・参戦回数（アーティスト別通算）・遠征情報（宿・交通）・当日のX投稿・グッズ購入記録・服装/参戦服 |
| 写真・半券 | 半券（電子チケットのスクショ含む）・グッズ・会場外観・フォトスポット |

- 根拠: mico's journal（チケットアルバム＝1ページ2枚収納＋セトリ・感想メモ欄、Notionで全参戦一覧＋個別ページにセトリ・当日ツイート紐付け）、マルマン推し活ノート特集（推し基本情報・年表・参戦日程/会場/感想・チケット/グッズ貼込）
- 市場: 推し活人口1,384万人・3.5兆円（CDG調査）○。財務省広報誌も若年層のデジタル課金定着を指摘○

### 4. 映画（劇場体験） ○

| 区分 | 項目 |
|---|---|
| 必須級 | タイトル・鑑賞日・**鑑賞方法（劇場/配信）**・評価 |
| あると嬉しい | 劇場名・スクリーン/座席・同行者・字幕/吹替・舞台挨拶等のイベント回 |
| 写真・半券 | 半券貼付は紙ノート文化の定番（コクヨ解説）。ポスター・ムビチケ |

- 根拠: Filmarks公式ヘルプ（スコア5段階0.1刻み・鑑賞日・鑑賞方法・自由レビュー）○、コクヨ映画記録ノートの書き方○
- 課金の証拠: Filmarksプレミアム月550円の中身は「鑑賞記録のビジュアライズ」＝記録の美しい可視化に支払い意思がある最重要ベンチマーク○

### 5. 食べ歩き・カフェ・居酒屋 ○

| 区分 | 項目 |
|---|---|
| 必須級 | 店名・訪問日・**注文メニュー**・場所 |
| あると嬉しい | 味の感想・店内の雰囲気・価格・再訪したいか・同行者 |
| 写真・半券 | 料理写真が主役・ショップカード貼付文化 |

- 根拠: カフェノート実践記事（訪問日・注文・味・雰囲気・写真・ショップカード）○、Notion公式にカフェ記録テンプレあり○

### 6. サウナ ○（王者あり・テンプレは低優先）

| 区分 | 項目 |
|---|---|
| 必須級 | 施設名・日付・感想 |
| あると嬉しい | サウナ温度・水風呂温度・混雑度・**サ飯/ドリンク**・セット数 |
| 写真 | 最大10枚（サウナイキタイ仕様） |

- 根拠: サウナイキタイのサ活公式仕様○。月間120万人利用・支援課金制度あり＝記録文化は最強だが**王者が既にいる**。フィールド設計（スペック＋体験の分離・1年まとめ）だけ借りる

### 7. 日本酒・ウイスキー・クラフトビール ○

| 区分 | 項目 |
|---|---|
| 必須級 | 銘柄・日付・評価・写真（ボトル/ラベル） |
| あると嬉しい | スペック（蒸溜所/酒蔵・年数・度数・精米歩合・酒米）・**香り→味わい→余韻の3段階官能メモ**・6軸フレーバーチャート（さけのわ型: 芳醇/華やか/穏やか/重厚/ドライ/軽快）・購入場所/店 |
| 写真 | ラベル写真が正本代わり（銘柄再購入の手がかり） |

- 根拠: Dear WHISKYテイスティングノート解説○、Sakenote・さけのわ・Untappdの項目○
- 隙間: Untappdは英語UI・SNS前提に不満あり＝「日本語で私的で美しい」の余地○。レーダーチャート資産が直撃するジャンル

### 8. 御朱印・寺社巡り ○（専用アプリ複数・優先度下げ）

- ホトカミ・神社がいいね・スマホ御朱印帳が既に強い（06-追加ジャンル発掘と同結論）
- 借りるべき設計: スマホ御朱印帳の「Wiki連携で寺社名から宗派・御本尊・創建年を自動補完」＝**場所メタデータの自動補完**という発想○
- favorecoでは「印」汎用フィールド（印の写真＋場所＋日付＋一言）で受ける

### 9. 温泉・宿 ○

| 区分 | 項目 |
|---|---|
| 必須級 | 施設名・訪問日・満足度・感想 |
| あると嬉しい | 泉質・湯の色・露天/内湯・宿泊/日帰り・料理・再訪意向 |
| UI示唆 | 「自分だけの温泉マップ」＝訪問地図の塗りつぶし達成感（meguri）。MapKit資産が活きる |

- 根拠: meguri・ユメグリの項目構成○

### 10. スポーツ観戦・プロレス・競馬 ○

| 区分 | 項目 |
|---|---|
| 必須級 | 大会/カード名・日付・会場・座席 |
| あると嬉しい | 対戦結果・スコア・同行者・（競馬）予想と回顧の分離・的中記録 |
| 写真 | 会場・スタメン/対戦カード表・グッズ |

- 根拠: 06-追加ジャンル発掘（LiveNote転用・umanote・自作リフィル文化）参照

### 11. 読書（受け皿） ○

- BOOTHの鑑賞記録テンプレ（カテゴリ・タイトル・ジャンル・鑑賞環境・評価・回数の6項目）が販売実績あり＝**テンプレに金を払う層の存在証明**○
- favorecoでは軽量記録（書名・読了日・評価・一言・表紙写真）で十分。読書メーターと戦わない

## ターゲット優先度への反映（01/06の結論とあわせた最終見立て）

1. **観劇・ミュージカル・2.5次元** — 唯一「検証済み」の実例があり、汎用手帳で代用中の未充足も確認。第1弾の中心で確定
2. **美術館・展覧会** — 半券+チラシ+作品リストのセット保存文化。決定版不在
3. **ライブ・フェス・推し活参戦** — 記録項目が最も多岐（セトリ・遠征・通算参戦数）。1,384万人市場。テーマパーク（06参照）もこの隣に置く
4. **映画（劇場体験）** — Filmarksの課金壁が「美しい記録への支払い意思」を証明済み。劇場体験特化で拾う
5. **酒（日本酒/ウイスキー/ビール）** — スペック+官能+レーダーの構造がMystorium資産と最も噛み合う。Untappd不満＝日本語×私的×美麗の隙間
6. 食べ歩き・温泉・スポーツ観戦 — テンプレは用意するが初期訴求の主役にはしない
7. サウナ・御朱印 — 王者あり。フィールド設計だけ参考にし、テンプレ優先度は最下位（自作カテゴリで対応可能）

## 未検証の論点（今後の宿題）

- 観劇以外のジャンルの項目は検索ベース。②-A仕様化の際に各出典を個別確認する
- ジャンル特化記録アプリのユーザー規模・継続率・課金実績の定量データ
- 「複数ジャンルを1冊で記録する」横断需要の一次証拠（半券分類ノート実践者は確認できたが、n=1の定性例）
- 半券のOCR読み取り・スキャン整理への需要
