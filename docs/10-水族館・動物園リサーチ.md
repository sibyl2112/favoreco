# 10 水族館・動物園 記録フィールドリサーチ

> 作成日: 2026-07-06
> 目的: 「おでかけ施設」テンプレ（施設種別サブレイヤー）の水族館/動物園部分のフィールド設計素材。
> 方法: deep-researchハーネス（5系統検索→ソース取得→3票反証検証→統合）。一次確認できたのはスイログ・Biomeの2アプリのみ。原ページは403で、App Store/Google Play/PR Times等の独立裏付けで復元。数値は点時点値のため**要確認**扱い。
> 関連: 06-追加ジャンル発掘リサーチ（テーマパーク・水族館の初出）／ 08 テンプレユニット設計

## 1. 既存サービスは2系統に分かれる

### 系統A：施設単位の訪問記録・全国制覇（＝スイログ）
- **スイログ**（iOS無料・個人開発）。全国**151箇所**（要確認）の水族館を対象に、**施設1km以内のGPSチェックイン**＋**過去訪問を写真の位置情報で手動チェックイン**して来訪を証明。スタンプラリー型。
- ゲーミフィケーション「**マイ水槽**」＝**訪問数に連動して水槽の生き物が増える演出**。→ **実際に見た「種」の同定でも個体識別でもない**（訪問数に紐づく演出にすぎない）。
- 出典: [開発者note](https://note.com/dancho0301/n/ndbf4cb1795df) / [App Store](https://apps.apple.com/jp/app/id6757411071)（確信度: medium・3-0）

### 系統B：野外自然観察のライフリスト・種同定（＝Biome）
- **Biome**。AIが撮影写真の場所・時期・形状から**日本のほぼ全動植物（約9.3万〜10万種・要確認）**を同定。写真投稿で**レア度連動ポイント／レベル／バッヂ**の「**いきものコレクション**」＝ゲーミフィケーションされたライフリスト。図鑑/マップ/SNS/クエストの複合。累計125万DL・1000万投稿超（要確認）。
- **ただし対象は野外自然**。水族館/動物園という**施設内観察は扱わない**。
- 出典: [Biome公式](https://biome.co.jp/app-biome/) / [App Store](https://apps.apple.com/jp/app/id1459658355)（確信度: high・3-0）

### 参考（未検証・要確認）
- **LINNÉ LENS**（かざすAI図鑑・日本の水族館対応をうたう）が検索に出たが一次確認できず。水族館内AI種同定の先行として**要追加調査**。

## 2. 決定的な空白（favorecoの狙い目）

> **施設訪問記録（系統A）と、種の博物学的観察記録（系統B）を橋渡しするサービスが存在しない。**

- スイログの「マイ水槽」は**見た種でも個体でもなく訪問数連動の演出**。
- Biomeは**野外自然が対象**で、水族館/動物園という施設内観察を扱わない。
- → **「水族館/動物園で実際に見た種・個体を、来訪ごとに累積していくライフリスト」は競合に存在しない**。これはユーザーの「博物学に属する」というインサイト（観察して記録する・生涯リスト・個体推し）と一致する空白領域。

## 4. favoreco設計への示唆

1. **競合の強み（スイログの施設コンプ／Biomeの野外種同定）は追わない**。favorecoは既存部品で「施設訪問＋見た種・個体の累積」を**両取り**する——これが空白。
2. **ライフリスト（見た生き物・動物の累積記録）＝差別化の核**。来訪(Visit)ごとに「見た種・個体」を記録し、対象(施設)や全体で累積して図鑑的に振り返る。U4リスト（呼び名=見た生き物/動物）を**回をまたいで累積する**方向に拡張する余地。
3. **施設訪問マップ**（訪問施設の地図塗り・スイログ的コンプ感）は、酒蔵県マップ・パーク訪問マップと同じ汎用マップ資産で成立。
4. **AI種同定は v1スコープ外**（外部/重い）。v1は「写真＋手入力で種・個体を残す」に留め、同定支援は将来（U16ラベルOCRと同じ半自動思想の延長）。
5. **個体推し・ショー・年パス行動は要確認**のまま、下地（自由に個体名・見たものを足せる器）だけ用意し、確証が出たら専用化する。
