> ステータス: 作成中（2026-07-06〜・1カテゴリずつ確定して追記）
> 位置づけ: ライブラリ画面の仕様。**カテゴリタブを切り替えると、そのジャンル専用の"エキスパート表示"になる**（画像比率も記録文化もジャンルで違うため、汎用グリッド1本にしない）＝favorecoの差別化。
> 関連: A3（フィールド）／A4（プリセット）／A5（集計・シリーズ・マップ・フィルモグラフィ）／A7（登録フロー）

## 0. 原則・共通ルール

- **ナビ**：5タブ（ホーム / カレンダー / ＋記録 / **ライブラリ** / マイページ）。ライブラリ上部に**カテゴリタブ**（すべて＋各柱・カテゴリ色）。
- **★ジャンルを「章」としてつなぐ（2026-07-16 確定）**：上部カテゴリ列のタップ、中央ジャンル名から開く全ジャンル一覧、Heroより下の本文横スワイプ、ページ末尾の前後ジャンルカードを同じジャンル切替へ接続する。切替時は内容と背景色を横方向へスライドさせ、単独ページの寄せ集めではなく「観劇の章→美術展の章→ライブの章」と地続きに見せる。
- **★ジェスチャー範囲（2026-07-16 確定）**：Hero/次の予定/最近行った、映画・書籍・おでかけ施設の共通ファーストビュー、御朱印Heroは、その内部で横カルーセルやタップ操作を使うためジャンル横スワイプの対象外。本文は最初の速度が縦より1.2倍を超えて横優勢な場合だけ方向限定パンを開始し、縦操作では横認識器を開始せず縦ScrollViewへ渡す。横パン開始後から終了100ms後までは本文タイルのNavigationLink/Buttonを無効化する。画面端24pt以内から始まる操作はシステム戻る操作等との競合を避けて無効化する。ドラッグ状態は本文コンテナへ閉じ込め、親のSnapshot/一覧を毎フレーム再計算しない。離した後は移動量または予測移動量で現在ページ/前後ページへ吸着し、約0.22秒の減速型アニメーションで切り替える。上部列/本文スワイプでは同じ縦位置を維持し、章末カードでは次章の先頭へ戻す。実機評価で同じ位置が不自然なら、先頭復帰へ変更できる余地を残す。
- **★長い一覧の読み込み（2026-07-16 確定）**：無限スクロールで全件を同時描画しない。初期表示は必要件数に絞り、ギャラリー、棚、一覧の続きはMystorium同様の「さらに表示」で段階展開する。表示が落ち着いた180ms後に前後ジャンルの先頭サムネイルだけをバックグラウンド生成し、キャッシュ済み画像は初回描画から同期表示する。章末の前後ジャンルは代表写真1枚と軽量集計だけを先読みする。
- **タブ切替＝別ビュー**：各ジャンルは「**既定ビュー＋見方切替（サブタブ）**」を持つ。汎用の一覧を全ジャンルで使い回さない。
- **「すべて」タブ**：比率が混ざるので**タイムライン/フィード**（横断・日付順）で受ける。
- **★per-genreカレンダーは作らない（2026-07-06 確定）**：日付で見る＝**グローバルの「カレンダー」タブ**（カテゴリのレイヤーON/OFFで各ジャンルだけ表示可）に一本化。各ジャンルのライブラリは**ギャラリー/グリッドに専念**（重複を避けシンプルに）。
- **並び替え・絞り込み・検索**：**各ジャンルの具体は後回し（2026-07-06 ユーザー指示）**。まず「既定ビュー＋見方切替」を各ジャンルで固め、フィルタ/ソートは後で一括設計する。
- **リピート**：ジャンルの流儀で（映画=日記に複数、酒=味覚マップに点、書籍=シリーズ、施設=通算）。
- **★スコープ（2026-07-06 明確化）＝本書は"見せ方（一覧の視覚レイアウト）"だけ**。「この監督のn本／このキャストn回」等の**横断カウントは統計＝A5（別画面）**。見せ方には含めない（例：日記・ギャラリー・マップ・棚は"並べ方"なので見せ方、"n回"は統計）。
- **設計思想（★2026-07-06 明確化）＝各ジャンルは専用アプリを"超える"のが目標**：映画がFilmarks/Letterboxdを超えたのと同じ公式を全ジャンルに適用＝**①見やすさを借りる ②専用アプリの弱点を突く ③横断（雑食）＋私的・美しさで上回る**。真似でなく超える。
  - 例外的に避けること：**DIYツール（Trello等）っぽいカンバンにしない**。状態管理も"記録の中に美しく溶かす"（下記 観劇のチケット状態）。
- **★ジャンルごとの色世界（2026-07-06 確定）**：**共通素材＝Liquid Glass／全体の基調＝白**。ただし**各ジャンルのライブラリは、そのジャンルのカテゴリ色を"世界観"としてまとえる**——映画＝**ダーク基調のガラス**（ポスターが映える）、観劇＝**ワインレッド＋ゴールド**（緞帳の赤・装飾の金＝劇場の世界観）、酒＝琥珀基調、御朱印＝朱基調…。白ベースは崩さないが、ジャンルタブに入ると色がつく＝雑食でも各柱に個性。エキスパート表示（レイアウト差）＋色世界（トーン差）の二層で差別化。

## 1. 映画【確定 2026-07-06】

**キャッチ：Filmarksの見やすさ＋Letterboxdの監督横断＋favorecoの私的・美しさ**

- **色世界＝クリーンな白ベース（2026-07-06 再確定）**：ダークキャンバス（背後薄暗）は"いまいち"で**不採用**。骨格も面も白/ガラス＝Filmarks的なクリーンさ。ポスターの色だけで十分映える。レイアウト＝**3列ポスター**。
- **構造＝1画面で「見たい(Interest)／見た(Watched)」に分割（Mystorium流・2026-07-06 確定）**：タブ切替でなく、**1画面に見たい（上・公開予定バッジ）と見た（下・★評価）の2セクションを縦に並べる**。**Journal（日記/時系列）は別モード**（上部で ライブラリ⇄Journal 切替）。
  - **見たい（Interest）**＝まだ観てない映画（記録の前段階・書籍の積読と同型）。TMDb検索で追加、未公開作は**公開日（日本）も取得**し「公開予定 7/10」バッジ。公開日は Coming Up・カレンダーに"予定"として流れ込む（チケット発売と同型）。観たら鑑賞日/評価を足して「見た」に昇格（Eventはそのまま・Visitが付く）。**自分の見たいリストの公開日だけ＝私的**（Trend browseは作らない・§要検討）。
- **ポスター下ステータスアイコン（2026-07-06 ユーザー要望）**：各ポスターの下に**自分のステータス**を小さく表示＆タップで切替＝**👁見た（緑）／＋興味あり（青）／★評価（見た時）**。Filmarksは"他人の数字"だが、favorecoは**自分の状態**（私的）。どのポスターからでも状態を一目＆その場で変えられる。
- **評価＝★5段階0.5刻み**（RESONANCE・全ジャンル統一・§9）。Journal（日記）は不採用（カレンダーと重複・§9）。
- **★"見たい/見た"の分割は汎用パターン**：Mystorium流の Interest/Done を1画面で分ける形は、書籍（積読/読了）・御朱印（気になる/参拝済）など各ジャンルに横展開できる。
- **★Hero Carousel（2026-07-08 確定）**：映画トップのファーストビューは「最近観た映画」固定にしない。基本は**次に観る映画**を出し、ユーザーが「トップに出す」で複数選んだ映画を横スワイプで切替できる。タップで映画詳細へ入る。
  - Hero候補：①手動でトップに出した映画 ②次に観る映画 ③もうすぐ公開 ④最近観た映画。
  - 状態ラベル：次に観る映画 / もうすぐ公開 / 最近観た映画 / 要対応（舞台挨拶・抽選等）。
  - Hero内の情報は最小限：大きいポスター、タイトル、日付/あと何日、状態チップ、★評価程度。監督・出演者・あらすじ・登録元・劇場/座席は詳細画面へ。
  - データ：`Event.isHeroCandidate` / `heroSortOrder` / `heroAddedAt` 相当を想定。1件固定ではなく複数候補。並びは予定日が近い順→追加日順（将来並び替え可）。
- **見せ方（視覚レイアウト）＝ポスターグリッド既定**：
  - 映画は言葉で語るよりポスターで思い出すジャンル。トップの一覧は**ポスターのみ/ポスター主役**を既定にする。説明文・監督・出演者・あらすじをカードに詰めない。
  - **共通表示モード（2026-07-17実装）**：3列ギャラリー／2列ミニ詳細／横長バナーを見出し右の3アイコンで切り替え、ジャンルごとに選択を記憶する。映画・観劇はギャラリー、ライブはバナー、書籍・その他はミニ詳細を初期値とする。御朱印専用トップは切替対象外。
  - **3列ギャラリー**：ジャンル基準比率の画像をaspect fillし、下へ評価と日付だけを置く。未記録は`—`、未来予定は日付を赤字にする。記録済みかつ予定ありは評価と赤い予定日を併記する。
  - **2列ミニ詳細**：小型Heroとして画像、タイトル、評価、最新記録日または予定日を表示する。
  - **横長バナー**：画像だけ正方形へaspect fillし、タイトル、日付、会場、チケット申込状態を最大3タグ表示する。
  - **チケット進捗（2026-07-17実装）**：進行中の申込がある場合だけ対象一覧の前へ表示する。複数公演・複数申込は上部の可変幅チップで選び、タップで詳細カード1枚を切り替える。日時、会場、申込経路、プレイガイド、任意タグもカード内の可変幅チップとして横スクロールできる。工程は抽選/先着に応じて2〜5個へ可変とし、工程名を丸内、日付を丸下へ置く。線は丸の外周間だけを結び、完了はジャンル色の実線、未来はグレーの点線、現在はジャンル色の二重丸にする。カード本体の横操作はジャンル遷移を維持し、横スクロールはチップ列だけに限定する。
  - 観たい映画は横スクロールのポスター列、鑑賞済みはジャンルタブ（すべて/アニメ/SF/ドラマ/ホラー/ドキュメンタリー等）＋ポスターグリッド＋下部「もっと見る」。
- **予定の扱い**：予定はグローバルのプラン/カレンダーで横断表示する。ただし映画トップでは、7日以内の鑑賞予定・公開日が近い観たい映画・舞台挨拶/抽選/入金/発券などの要対応がある場合のみHero/小Attentionに表示する。予定が無ければ出さず、ポスターグリッドを優先。
- **佇まい**：白/ガラス基調に、ポスターの色を主役にする。映画ページだけ全画面ダークにはしない。
- **参照**：Letterboxd（ダーク・密ポスターウォール・Journal）／Filmarks（ポスターグリッド）。
- ※「この監督/俳優 n本」の横断は**統計＝A5**（別画面）。フィルタ/並び替えも後回し。

## 2. 観劇【表示方向 確定 2026-07-06／フィルタは後回し】

**ゴール：シアティ（Theaty）を超える**（真似でなく上回る）。

- **色世界＝ワインレッド＋ゴールド（2026-07-06 確定）**：緞帳の赤・装飾の金＝劇場の世界観。映画のダークと好対照。Liquid Glass素材にワインレッド／ゴールドのトーン。
- **見せ方（視覚レイアウト）★2026-07-06 確定（カレンダー切替は廃止）**：
  - **ライブラリ＝ギャラリー（チラシ）のみ**。「いつ観たか」で見たいときは**グローバルの「カレンダー」タブ（観劇レイヤーON）**に任せる。**per-genreカレンダーは重複＝廃止**（映画と同じ構造：ライブラリ＝グリッド／日付＝カレンダータブに一本化）。★2026-07-06 ユーザー指摘で修正。
  - **観たい / 観た 分割**（Mystorium流）＋**チラシ下ステータスアイコン**（🔖観たい / 👁観た / ★評価）＋**観たいにチケット状態バッジ**（当落待ち等）。
  - **チケット要対応ストリップ**（当落待ち/入金待ち/発券待ち）を上部に常時表示（記録に溶かす・Trelloカンバンにしない）。
  - **密度＝3列（既定・Theaty風の詰め）／2列（切替オプション）**。**上質さは密度でなく色世界（ワインレッド＋ゴールド・クリーム地＋金枠チラシ）と質感で出す**。チラシ＝A4縦。
- **トップ構成（2026-07-08確定）**：
  ```text
  Hero（次に観る / 要対応がある公演 / 最近観た / 手動固定）
  チケット要対応
  申込・観劇予定
  観劇ライブラリ（縦スタック）
  データ
  ```
- **Hero**：次に観る公演を優先。なければ要対応がある公演、最近観た公演、手動固定の順。チラシ/キービジュアル、公演名、劇場、日付、開演、チケット状態、座席（任意）を表示。
- **チケット要対応**：受付開始/申込締切/当落/入金/発券など、忘れると体験を失う進行中の申込だけを表示する。複数公演・複数申込は公演選択チップで切り替え、トップには選択中の詳細カード1枚だけを出す。`すべて見る`でチケット一覧へ遷移し、対象がない時は畳む。
- **申込・観劇予定**：まだ完了していない未来/進行中の公演回だけを表示する。気になる/申込前/当落待ち/入金待ち/発券待ち/観劇予定が対象。観劇済みになったものはここから外し、観劇ライブラリ/観劇履歴へ移る。
- **観劇ライブラリ**：観劇トップの主役。横スクロールや独立リピート欄ではなく、作品カードを縦にスタックする。チラシ＋タイトル＋最終観劇日＋★評価＋`3回観劇`などのリピート回数を表示。遠征/複数会場がある場合は `赤羽・大阪 ほか` のように短く出す。
- **観劇回詳細**：上部はジャンルトップの次の予定カードと同じ左ポスター＋右情報の構成を、通常詳細より左右8ptずつ広げて使う。B判ポスター、公演名、観劇日時、会場、座席、評価を表示し、右下へ`記録を編集`と`…`を置く。`…`内は確認後にこの観劇回だけを削除する。入力済みキャストはHero直下へ横スクロール表示し、人物写真がない場合は名前の先頭1文字を円内へ表示する。演出・脚本など出演系以外の人物は`スタッフ・関係者`として下段へ残す。
- **リピート観劇**：独立セクションは作らない。観劇ライブラリの作品カード内メタ情報として `n回観劇` `最多リピート` などを出す。
- **データ**：トップのマストは `今年の観劇回数`。次点で `申込中/要対応チケット数`、`最多リピート作品/回数`。`劇場数`、`マチソワ回数/比率`、`よく観る俳優`、`よく行く劇場`、`ジャンル比率`、`チケット当選率`は詳細統計側を基本にし、トップ表示は設定/おすすめで差し替え可能にする。
- **トップに出しすぎないもの**：チケットサイトアカウント、パスワード、全申込履歴、落選一覧、名義別当選率、長文感想、キャスト詳細は詳細/統計/設定へ逃がす。
- **佇まい**：詰め気味の3列＋ワインレッド/ゴールドの劇場感（全画面ダークにはせず、共通骨格＝クリーム地に金枠チラシ）。
- **チケット状態＝"美しく統合"（Trello自作カンバンにしない）**：
  - 上部に**「要対応」ストリップ**：`本日12:00申込開始` `申込締切 今日23:59` `📩当落 明日` `🔴入金待ち2` `🟡発券待ち1`（期限つき・タップで該当公演へ）＝一目で"今やること"（※これは"見せ方"上の機能表示。件数は状態の数であって集計統計ではない）
  - トップに出すのは **申込開始（特に当日）/ 申込締切 / 当落発表 / 入金締切 / 発券開始・発券待ち / 公演当日** のみ。落選履歴・過去申込・名義別当選率・アカウント情報は詳細/統計/設定へ逃がす。
  - 要対応カードには `申込ページを開く` を置く。`アカウント情報` を開く場合は Face ID / Touch ID / 端末パスコードで認証して、IDコピー・パスワードコピー（Keychain）を行う。
  - 各公演のチラシに**小さな状態バッジ**（色ドット）
  - → 記録（チラシウォール）と状態が**1画面に溶ける**。Trelloの分断も、Theatyの"状態管理なし"も超える
- **超えどころ**：チケット状態の統合／横断（雑食）／チラシの美しさ。
- **参照**：シアティ（座席・感想無制限・思い出コレクション・カレンダーで観劇ログ／単一ジャンル・状態管理なし）。
- ※キャストで束ねる横断は**統計＝A5**。フィルタ/並び替えは後回し。

## 3. 美術展【表示方向 確定 2026-07-06／フィルタは後回し】

**ゴール：Punct Diary を超える。**

- **見せ方（視覚レイアウト）＝縦長ギャラリー（観劇と同系・ゆったり）＋作品ギャラリー**：
  - **既定＝ゆったり大きめの縦長ギャラリー（A4チラシ・1〜2列）**（観劇と同じ佇まい・展覧会も"一つずつ味わう"）
  - **心に残った作品ギャラリー**（U4リスト＝見た作品を写真＋メモを敷き詰めた**作品ギャラリー**）★Punct Diaryの"作品ごと記録"を、より美しいギャラリーに昇華＝美術展固有の見せ方
  - **会期・予定Attention＝ミニガントバー型（2026-07-08確定）**。会期終了間近・日時指定・チケット済みをHero下に最大3件表示する。
    ```text
    静寂の色彩
    青凪美術館
    4/20 ━━━━━━━●━━ 7/15  あと7日
    [気になる] [終了間近]
    ```
  - 会期バーは、横棒＝会期、●＝今日位置、残り日数＝右側に表示。終了間近は赤/オレンジ、予定あり/チケット済みはチップで表示。タップで展示詳細、もっと見るで会期一覧/グローバルカレンダーへ。
  - Heroは動的：会期終了が近い気になる展示 / 次に行く展示 / トップに出す展示 / 最近行った展示。美術展は「気づいたら終わっていた」を避けるため、予定・会期終了の優先度を高くする。

- **構造＝帳面（入れ物）→印（中身）の2階層**（GoshuinBookエンティティ・spec-A3）。ただしトップHeroの主役は帳面表紙ではなく、**最近いただいた印そのもの**。
- **印種別タブ**：`すべて / 御朱印 / 御城印 / 御船印`。内部は同じ「印の記録」だが、表示ラベルと場所種別を切り替える。御朱印が主役、御城印/御船印は同じ構造で軽量に受ける。
  - 御朱印：場所=寺社、帳面=御朱印帳、気になる=行きたい寺社。
  - 御城印：場所=城、帳面=御城印帳、気になる=行きたい城。
  - 御船印：場所=港・船、帳面=御船印帳、気になる=行きたい港・船。
- **Hero＝最近いただいた印**：
  - 表示：印写真 / 種別 / 場所名 / 日付 / 都道府県 / 帳面名 / 直書き・書き置き（御朱印時）。
  - 選定：①手動でトップに出した印 ②最近いただいた印 ③同じ季節/去年の印 ④お気に入り印 ⑤写真が綺麗な印。
  - タップで印の詳細へ。
- **最近いただいた**：印写真主役のカード。場所名・日付・都道府県のみ最小表示。選択中の印種別タブで絞る。
- **帳面**：主役ではなく整理単位。最新帳だけ少し大きめ、帳面一覧は小さめカード。表紙写真があれば出す。なければ淡いプレースホルダー。収録数・最新記録を表示。
- **御朱印帳の縦横比**：通常（縦16×横11cm）／大判（縦18×横12cm）／横長（182×257mm）。表紙写真・裏表紙写真を保存可。
- **帳面展開**：タップで中の印を表示。iPhone縦の見え方で、縦展開/全画面"帳を開く"ビューを検討。最新帳は既定選択。

## 8. 書籍【表示方向 確定 2026-07-08／フィルタは後回し】

**ゴール：読書管理アプリの重さに寄せず、"買った/気になった本を忘れない"ことと、表紙が積み上がる気持ちよさを両立する。**

- **色世界＝白/アイボリー＋控えめなブラウン**。棚板・木目・古い"本棚チック"な表現は使わない。Liquid Glassのカードと余白で、表紙が主役になる現代的なライブラリにする。
- **状態設計＝3レイヤー**：
  - **共通興味ステータス**：気になる / 欲しい（映画の観たい、観劇・ライブの行きたい、酒の飲みたいと同じレイヤー。表示ラベルだけ書籍向けに変える）。
  - **書籍固有の入手状態**：未入手 / 購入済み。買ったか忘れる問題を解くため、購入済みはトップでも見える。
  - **購入済み後の任意読書状態**：未読 / 読書中 / 読了 / 中断 / 再読中 / 再読了。購入済みになったら表示されるオプションで、入力必須にしない。
- **書影の扱い**：
  - ISBN/URLから取得した書影は、外部URLだけに依存せず、アイキャッチ用にローカル保存/キャッシュする。オフラインでも表示できる。
  - 登録・編集時に本の種類をコミック`112×174mm`、新書`103×182mm`、ハードカバー`127×188mm`、文庫`105×148mm`、雑誌`182×257mm`、写真集・画集`210×297mm`から選ぶ。選択した実寸比率を作品と記録へ保存し、ジャンルトップ、Home、一覧、対象/記録詳細、入力サムネイルのアイキャッチ枠へ共通適用する。
  - 書影は選択した比率枠を中央aspect fillで隙間なく覆い、超過部分をトリミングする。新規の既定はハードカバー、既存の種類未選択データは従来の`1:1.45`を維持する。横長バナー表示だけは共通仕様どおり正方形へ切り抜く。
  - `remoteImageURL` は再取得用に保持。`eyecatchPath` は保存済み代表画像。写真欄へは自動追加しない。
  - ユーザーが「写真にも保存」を選んだ場合のみ、通常写真またはコレクションへ追加する。自分で撮った本の写真は通常写真として登録し、アイキャッチにも設定可。
- **登録元（ISBNから登録 / URL保存 / 写真から登録）はトップに出さない**。詳細画面の登録履歴/メタ情報で表示する。トップは表紙・状態・最低限の文脈に絞る。
- **著者/訳者/出版社の表示**：
  - トップカードでは著者を1行まで（必要な表示モードのみ）。訳者・出版社・ISBNは詳細画面へ回す。
  - 画面の主役は表紙。テキスト過多にしない。
- **メインページ構成**：
  1. **最近買った本**：直近で購入済みにした本。表紙＋タイトル＋購入済みチップ＋購入日。読書状態は任意チップ。
  2. **読書データ**：今月買った / 今年買った / 読書中 / 気になる。下部カードで月間/年間の購入・読了推移、ジャンル比率。
  3. **気になる本**：書店でISBN登録、URL登録、手入力で放り込んだ未購入候補。横スクロール。登録元は非表示。
  4. **購入済み一覧**：ジャンルタブ（すべて / 小説 / 漫画 / 技術書 / 実用書 / デザイン）で切替。下部に**「もっと見る」**を置く（右上のすべて見ると重複させない）。
  5. **読書メモ**：私的メモ、心に残った一文、写真メモの最新。本文は短く、詳細で展開。写真枚数（例：写真5枚）を表示可。
  6. **シリーズ/巻数**：必要に応じて別セクション。長編漫画・シリーズ本はスタック表示/巻スクロール。
- **タイル表示モード**：
  - **書影グリッド（既定）**：表紙だけを主役にする。表紙＋小さな状態チップ（購入済み/未読/読書中/読了/写真あり）程度。最も美しく眺めるモード。
  - **情報カード**：表紙＋タイトル＋著者1行＋状態＋購入日/読書状態。買った/読んだ確認をしたい時の実用モード。
  - **リスト**：大量管理用。タイトル・著者・状態をテキスト中心で並べる。
  - 表示モードはジャンル内で切替可能。初期表示は書影グリッド寄り。
- **読書量管理**：
  - 月間/年間の購入冊数・読了冊数・読書中冊数・気になる冊数・ジャンル比率を表示する。
  - 読書タイマー、連続読書、毎日の目標は読書習慣アプリ寄りなのでv1では主役にしない。
  - カレンダーには購入日・読始日・読了日・メモ日を表示できる（グローバルカレンダー側でレイヤー表示）。
- **読書メモ/写真**：
  - 読書メモ、心に残った一文、写真メモは採用。公開レビューではなく私的メモ。
  - ページ写真・引用OCRは任意。シェアカードに引用を出す場合は短文に制限する。
- **差別化**：読書量管理を作り込みすぎず、favorecoらしく「気になった本→買った本→読書メモ/写真→振り返り」を軽く残す。Bookmory的な高機能ノート/タイマーではなく、表紙主役の私的アーカイブに寄せる。

## 9. 共通の見せ方機能（全ジャンル・2026-07-06 確定）

Letterboxdの売りから採用した、ジャンルをまたぐ見せ方：

- **Lists（マイベスト・自作リスト）**：ユーザーが**順位つき/テーマ別のリスト**を作れる（「マイベスト観劇10」「今年の本」等）。ジャンル内でも横断でも。眺めて嬉しい＋好みが結晶する。※ソーシャル共有はしない（私的）。
- **推しピン（Four Favorites）**：各ジャンル（や全体）の**推しを大きく固定表示**＝favorecoの「柱」思想の顔。プロフィール/カテゴリ上部に。
- **ヒートマップ（体験頻度）→ 年間まとめ(A5)**：GitHub草グラフ的に体験の頻度を色濃淡で。年間まとめ（買い切り）の顔に。
- **評価＝★5段階0.5刻み（RESONANCE）で全ジャンル統一**（映画も）。100点/10点は消費者標準でなく迷いやすいため不採用（映画だけ"100点詳細モード"を望むならオプション余地・既定は★）。※Letterboxd/映画.com/Filmarksも5段階系。
- **Diary（時系列日記）は不採用**（カレンダーと重複・シンプル優先。2026-07-06）。
- **ジャンルトップのセクション順カスタム**：全ジャンルで、トップ内セクションの並び替え・表示/非表示を持つ。例：Hero / 気になる / 記録一覧 / データ / メモ / コレクション。完全自由レイアウトはやらない。`required`（要対応/同期エラー等）・`recommended`（Hero/最近の記録）・`optional`（データ/メモ等）に分け、requiredは非表示不可。
