# コンセプトと勝ち筋（批判的分析込み）

> 2026-07-04 時点の戦略仮説。Mystorium（謎解き・イマーシブ記録アプリ）から派生する汎用「好き」記録アプリの検討正本。
> 前提資料: `01-競合分析.md`

---

## 結論

**「書籍・映画も入る汎用アプリ」ではなく「作品DBが要らない"体験"の記録アプリ」に絞る。**

- 書籍・映画は読書メーター/Filmarks/Letterboxd の DB・レビュー資産に勝てない
- 美術展・観劇・ライブ・食べ歩きは「自分が行った、その日、その席、その写真」が主役 ＝ 外部DB不要 ＝ Mystorium と同じ構造 ＝ 個人開発の土俵
- 書籍・映画は「入れられるが主役ではない」位置づけに留める

## ポジショニング一文（案）

> **観た・行った・味わったを、美しく一生残す。サブスクなし。**

- サブスク疲れ（Day One 離反・Sofa への買い切り要望）へのカウンター
- 統計課金（Filmarks月550円・Letterboxd Pro）へのカウンター
- 「一度買えば一生」は Mystorium の建国の魂の継承

## 勝ち筋（4点）

### 1. 敵の課金壁が、こちらの買い切りの武器になる
Filmarks は「自分の記録のビジュアライズ」を月550円に、Letterboxd Pro（$19/年）も中核価値は統計。**「自分の記録を美しく見る」に支払い意思があることを競合が証明済み**。それを買い切りに全部入れる。

### 2. 「体験記録」は作品DBが要らない＝個人開発の土俵
技術的には Mystorium の `EventCategory` enum をユーザー定義データに一般化するだけで到達できる（詳細は `03-Mystoriumからの技術移行メモ.md`）。

### 3. 観劇クラスタが最短の橋
イマーシブ勢と観劇勢はユーザーが重なる。観劇特有ニーズ（座席番号・マチソワ・キャスト変更・複数回観劇）は Mystorium のリピートグループの延長線。既存ユーザーの口コミ（X）が直接効く唯一のカテゴリ。**初期市場はここ＋美術展**。

### 4. バイラル装置＝シェアカード
「年間ベスト」「観劇記録カード」は X で回る型。最初から設計の中心に置く（Mystorium の RESONANCE カード資産を流用）。

> **★2026-07-07 決定：SNS（公開型レーティング・口コミ・フォロー）は非採用。「見せたい」欲はシェアカードで満たす。**
> - 理由：公開UGCはサーバ/共有DB・アカウント・**モデレーション＋App Store Guideline 1.2（UGC義務）**・コールドスタート・**プライバシー方針（自分の・私的な記録）との矛盾**を丸ごと抱える＝別プロダクト級の負担。ネットワーク効果を持つ既存勢（Letterboxd/食べログ/Untappd）と最も勝ちにくい土俵で殴り合うことになる。
> - **コミュニティ集計は"作った人から借りる"**：さけのわ（日本酒版Untappd＝本格バックエンド運用）の**公開API（さけのわData・読み取り専用）**で集計フレーバー/ランキングを"参考値"として取得。映画=TMDb、書籍=openBD/NDL も同型。**大変なバックエンドは他社が運用済み、favorecoはその出力だけ読み取りで使う**。個々の口コミ文の転載は規約・権利上しない（集計値の参考表示に留める）。
> - favoreco本体は **自分の記録 × 横断 × 登録の楽さ** に集中。"社交"はシェアカード（既存SNSへ吐き出す・バックエンド不要）で代替。将来サブスクの目玉に限定的共有を検討する余地は残すが、v1〜当面はSNSを持たない。

## 対象カテゴリ（初期テンプレート案）

**コア（体験記録・DB不要）**: 美術館・博物館 / 演劇・観劇 / ライブ・フェス参戦 / 食べ歩き・飲み歩き（居酒屋・喫茶店） / 映画（劇場体験として）
**追加候補**: 御朱印・寺社巡り / サウナ / 日本酒・ウイスキー・クラフトビール / スポーツ観戦 / 庭園・建築・聖地巡礼 / 温泉・宿 / プラネタリウム
**受け皿**: 書籍・その他（自作カテゴリ）

## 批判的視点（正直なリスク）

1. **「汎用」は誰の一番にもならない**。読書はメーターに、映画は Filmarks に残る。実際に使われるのは2〜3カテゴリ → 最初から「体験記録」に照準を絞るのが誠実。
2. **最大の競合はアプリではなく Notion と「記録しない」こと**。機能では Notion に勝てない。勝てるのは「開くたびに嬉しい」体験の質だけ。
3. **キログという先行者**（なんでも記録・買い切り）。機能表ではなく世界観で差別化する前提なら共存可能。
4. **推し活はレッドオーシャン化が始まっている**（参戦記録アプリが2023〜25年に複数）。単発機能では埋もれる。ジャンル横断の統合記録で勝負。
5. **買い切りの経済性**。継続開発費を賄うには相当なDL数が必要。CloudKit のサーバー費ゼロ構造は防御になるが、収益の天井は低い。創設メンバー型の初速設計＋シェアカード露出が生命線。

## 売り込み方

- **初期市場**: 観劇・美術展クラスタ（手薄 × Mystorium ユーザーと重なる × X で口コミが回る文化）
- **データの信頼を明文の売りに**: エクスポート自由・人質にしない（読書メーターCSV不可などへの不満は実在）
- **開発ストーリー発信**: 「サブスクが嫌で買い切りで作った」はそれ自体がコンテンツ（note/X・個人開発界隈＋趣味界隈の両方に刺さる）
- **価格**: Mystorium より強気でよい（Day One 日本 ¥2,800/年、Sofa $29.99/年が相場アンカー）。具体額は未定 — Mystorium で確定した「差額アップグレード（頭金方式）」「月額:年額=2:1」「創設メンバー特典」の型を流用検討

## Mystorium との関係（確定済み方針の再掲）

- 派生アプリ＝**長期の主力（フラグシップ）**。Mystorium はニッチ/プロ版として尖りを保ち永続
- 2アプリは兄弟として並走。実用記録は汎用アプリ、謎解き・イマーシブの深い記録は Mystorium という併用も想定
