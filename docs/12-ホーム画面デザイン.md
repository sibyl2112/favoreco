## 2. デザイン言語 → DESIGN.md に統合（差分のみ記載）

トークンは **`docs/DESIGN.md` が正本**。本書は再定義しない。以下は3案モックと DESIGN.md の**差分＝要調整点**。

| 項目 | 3案モック（本書の初稿） | DESIGN.md（正本） | 判定 |
|---|---|---|---|
| アプリ地（ホーム） | 温かいクリーム `#F3ECE0` | 寒色白 `#f5f6f9`（app chrome＝ニュートラル） | **要決定**（暖色 vs 寒色白） |
| アクセント（ホーム） | 真鍮ゴールド `#B08D57`（＋ボタン・タブ） | 青 `#3a6df0`（app_accent）。金はジャンル accent | **要決定**（金 vs 青） |
| カード | 白＋やわらか影 | 半透明 Liquid Glass＋blur＋スペキュラ（§5-2） | DESIGN.md採用（モックを寄せる） |
| カテゴリ色 | 独自に暖色寄り5色 | ジャンル8色世界（§3・観劇ワイン/映画チャコール等） | **DESIGN.md採用**（本書の独自色は破棄） |
| アイコン | インラインSVG細線（可） | SF Symbols（実装）／SVG細線（モック）＝一致 | OK |

**核心の対立**: ユーザーが「テンション爆上がる」と評価した3案は**暖色クリーム＋金でホーム全体**。DESIGN.md 憲法#4は**ホームはニュートラル、色はライブラリだけ**。→ ①ホームもニュートラル白＋青に寄せる（憲法維持）か、②ホームだけ暖色＋金を"favorecoの顔"として例外許容し憲法を改訂するか。**§5-0で要決定**。

### 2.2 タイポグラフィ
- **ワードマーク・見出し・日付・タイトル＝セリフ（明朝／Didone系）**。Mystorium の Cinzel（装飾的）とは別人格の、editorial で端正なセリフ。
- **UI・ラベル・本文＝サンセリフ（Hiragino Sans / SF）**。
- 英字の小ラベル（TODAY / YOUR JOURNEY / SOMEDAY / ATTENTION）は**トラッキング広め・大文字**でリズムを作る。

### 2.3 素材・トーン
- リキッドグラス基調: **浮遊タブバー**は半透明ガラス（backdrop-blur）。カードは白＋やわらかい影（`0 8px 24px rgba(70,55,35,.10)`）。
- 角丸: カード 18–22px／写真 12–14px／タブバー 26px。
- 写真は必ず主役に。カテゴリはガラスの小タグを写真左上に載せる。

## 3. ホーム画面の構成（統合案＝モックの並び）

上から: **ヘッダー → TODAY ヒーロー → 要チェック → YOUR JOURNEY → 今月の体験（カウンター）→ 今月の記録（カテゴリ別）→ 最近の記録 → SOMEDAY**。

- **ヘッダー**: `Favoreco` セリフ＋サブコピー「好きな体験を、人生の軌跡に。」＋通知ベル＋アバター（案3ベース）。
- **TODAY ヒーロー**: 日付ブロック＋ポスター＋カテゴリchip＋会場＋チケット状態＋「詳細を見る」。横スワイプで次の予定を切替（ドット）。
- **要チェック**: 支払い期限・抽選結果など「締切のあるもの」だけをバッジ数付きで。案1/2 の ATTENTION を絞り込んだ形。
- **YOUR JOURNEY（目玉）**: 1つの体験の**ライフサイクル**を縦タイムラインで。発見 → 抽選申込 → 当選・確保 → **体験する（現在地・ゴールドの現在ノード）** → 記録する。favoreco 独自の「体験を計画から記録まで見届ける」世界観をホームで可視化する装置。
- **今月の体験**: 体験予定／体験済み／記録した／お気に入り の4カウンター（案3）。
- **今月の記録**: カテゴリ別カウンター（彩色アイコン・案2）。※上の4カウンターと役割が重複気味 → §5 で要判断。
- **最近の記録 / SOMEDAY**: 写真カードの横スクロール。SOMEDAY は縦長ポスター比率で「観たい/行きたい」を掻き立てる。

タブ: モックは **ホーム / プラン / ＋ / カレンダー / ライブラリ**（3案共通）。⚠️ **spec-B1 §0 の確定ナビは「ホーム / カレンダー / ＋記録 / ライブラリ / マイページ」**（5タブ・「プラン」なし・「マイページ」あり）。→ ナビは spec-B1 に合わせる（マイページ＝登録情報/連携ハブ A8 §3.4 の入口）。中央＋の色は §2 の金/青決定に従う。

## 4. データとの整合（既存仕様との接続）

- ヒーロー・YOUR JOURNEY は Event/Visit と直結（spec-A1）。「発見/申込/当選/体験/記録」は **spec-A8 のチケット状態pipeline**（気になる→申込前→当落待ち→当選→入金待ち→発券待ち→参戦済）＋登録フロー（spec-A7）が発生源。要チェック＝spec-A8 の「要対応」（締切）＋通知。
- 「体験予定/体験済み/記録した」は Visit の状態、カテゴリ別カウントは RecordCategory 別集計（spec-A5 想定）。
- SOMEDAY ＝「いつか行きたい」＝体験前の Event（未来日・チケット未確保）。プラン画面とも連動。
- 写真は PhotoBlob（09）。カードのアイキャッチは Visit.eyecatchPath。

## 5. 要選択・未決事項（ユーザー判断）

0. **★最重要：ホームの色（暖色金 vs ニュートラル青）**。3案（暖色クリーム＋金でホーム全体）と DESIGN.md 憲法#4（ホームはニュートラル白＋青・色はライブラリだけ）が対立。選択肢＝(A)憲法通りホームは白＋青に寄せる／(B)**ホームだけ**暖色＋金を"favorecoの顔"として憲法の明示的例外にし DESIGN.md を改訂。→ 決定を DESIGN.md に反映してからモックを直す。
1. **ナビ**: spec-B1 の5タブ（ホーム/カレンダー/＋/ライブラリ/マイページ）に統一（モックの「プラン」を廃し「マイページ」へ）。
2. **集計の重複**: 「今月の体験（4カウンター）」と「今月の記録（カテゴリ別）」を両方出すか、片方に統合するか。→ モックは両方併記。おすすめは **YOUR JOURNEY の直後に片方**（体験の物語 → 数字は1ブロックに集約）。
3. **YOUR JOURNEY の対象**: 直近1件の体験を追う（モック）か、複数体験の動きを時系列マージするか（案3は後者寄り）。
4. **ヒーローの見せ方**: 案3の白カード型（上品）か、案1の暗色大判キービジュ型（没入・テンション高）か。→ 「テンション爆上がる」重視なら案1寄せも検討。
5. **書体**: DESIGN.md がジャンルで serif/sans を規定（app chrome＝sans）。ホームのワードマークだけ serif 例外にするかは §0 の色決定とセットで。
6. **サブコピー**: 「好きな体験を、人生の軌跡に。」（案3）／「あなたの『好き』を、記録に残そう。」（案1）のどちらを採用するか。正式キャッチは「観た・行った・体験したを、美しく一生残す。」（concept 確定）なので、ホームのサブは短い方を。
