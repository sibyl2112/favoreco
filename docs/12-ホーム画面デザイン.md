> **役割**: コンセプト §6 で「別途デザイン仕様で確定」と保留していた**具体のデザイン言語（フォント・カラー・トーン）とホーム画面の構成**を、ChatGPT のインスピレーション3案から抽出・確定していくための作業ドキュメント。
> **ステータス**: 草案（デザイン言語＝ほぼ確定候補／ホーム構成＝要選択）

## 1. インプット（ChatGPT 3案）

3枚とも favoreco ホーム画面。共通して「白カード on クリーム地・写真が主役・真鍮ゴールドの差し色・セリフのワードマーク・浮遊タブバー＋中央＋」。コンセプト §6（白ベース・リキッドグラス・写真主役）と完全に地続き。

| | 案1 | 案2 | 案3 |
|---|---|---|---|
| 地色 | クリーム地・フラット | クリーム地＋白カード | クリーム地＋白カード（最も白い） |
| ヒーロー | 暗色の大判キービジュ（演劇ハムレット） | 白カード＋ポスター（映画） | 白カード＋日付ブロック＋ポスター（ミュージカル） |
| カテゴリ集計 | 線画アイコン＋数字（5列） | **彩色**アイコン＋数字（5列） | 4カウンター（体験予定/済/記録/お気に入り） |
| 注目枠 | ATTENTION（3カード） | ATTENTION（枠線3カード） | 要チェック（2件・バッジ数） |
| 目玉 | ─ | ─ | **YOUR JOURNEY（体験のライフサイクルのタイムライン）** |
| ワードマーク | 太めセリフ | 細めセリフ | 細めセリフ＋サブコピー＋アバター |

**ユーザー評価**: 「アプリを開くたびにテンション爆上がる構成が最高にいい」＝**密度と物語性**（次の予定→注目→軌跡→集計→記録の縦の流れ）が刺さっている。

## 2. デザイン言語（抽出・確定候補）

### 2.1 カラー
| トークン | 値 | 用途 |
|---|---|---|
| bg | `#F3ECE0` | 温かいクリーム地（アプリ地） |
| card | `#FFFFFF` | カード面（写真とテキストの器） |
| ink / ink-2 / ink-3 | `#2C2823` / `#8B8378` / `#B4AC9F` | 本文／サブ／極淡 |
| **gold（真鍮）** | `#B08D57` | 差し色。＋ボタン・強調・アクティブタブ・アクセント |
| line | `#EDE6DA` | 罫線・区切り |

カテゴリ色（案2の彩色アイコン由来・くすませて上品に）: 映画`#A98B4E`／演劇`#B5715A`／美術展`#7E9078`／酒`#8E7E96`／御朱印`#C1583F`。

### 2.2 タイポグラフィ
- **ワードマーク・見出し・日付・タイトル＝セリフ（明朝／Didone系）**。Mystorium の Cinzel（装飾的）とは別人格の、editorial で端正なセリフ。
- **UI・ラベル・本文＝サンセリフ（Hiragino Sans / SF）**。
- 英字の小ラベル（TODAY / YOUR JOURNEY / SOMEDAY / ATTENTION）は**トラッキング広め・大文字**でリズムを作る。

### 2.3 素材・トーン
- リキッドグラス基調: **浮遊タブバー**は半透明ガラス（backdrop-blur）。カードは白＋やわらかい影（`0 8px 24px rgba(70,55,35,.10)`）。
- 角丸: カード 18–22px／写真 12–14px／タブバー 26px。
- 写真は必ず主役に。カテゴリはガラスの小タグを写真左上に載せる。

## 3. ホーム画面の構成（統合案＝モックの並び）

上から: **ヘッダー → TODAY ヒーロー → 要チェック → YOUR JOURNEY → 今月の体験（カウンター）→ 今月の記録（カテゴリ別）→ 最近の記録 → SOMEDAY**。

- **ヘッダー**: `Favoreco` セリフ＋サブコピー「好きな体験を、人生の軌跡に。」＋通知ベル＋アバター（案3ベース）。
- **TODAY ヒーロー**: 日付ブロック＋ポスター＋カテゴリchip＋会場＋チケット状態＋「詳細を見る」。横スワイプで次の予定を切替（ドット）。
- **要チェック**: 支払い期限・抽選結果など「締切のあるもの」だけをバッジ数付きで。案1/2 の ATTENTION を絞り込んだ形。
- **YOUR JOURNEY（目玉）**: 1つの体験の**ライフサイクル**を縦タイムラインで。発見 → 抽選申込 → 当選・確保 → **体験する（現在地・ゴールドの現在ノード）** → 記録する。favoreco 独自の「体験を計画から記録まで見届ける」世界観をホームで可視化する装置。
- **今月の体験**: 体験予定／体験済み／記録した／お気に入り の4カウンター（案3）。
- **今月の記録**: カテゴリ別カウンター（彩色アイコン・案2）。※上の4カウンターと役割が重複気味 → §5 で要判断。
- **最近の記録 / SOMEDAY**: 写真カードの横スクロール。SOMEDAY は縦長ポスター比率で「観たい/行きたい」を掻き立てる。

タブ: **ホーム / プラン / ＋ / カレンダー / ライブラリ**（3案共通）。中央＋＝真鍮ゴールドの登録ボタン。

## 4. データとの整合（既存仕様との接続）

- ヒーロー・YOUR JOURNEY は Event/Visit と直結（spec-A1）。「発見/申込/当選/体験/記録」はチケットユニット・登録フロー（spec-A7）の各段が発生源。
- 「体験予定/体験済み/記録した」は Visit の状態、カテゴリ別カウントは RecordCategory 別集計（spec-A5 想定）。
- SOMEDAY ＝「いつか行きたい」＝体験前の Event（未来日・チケット未確保）。プラン画面とも連動。
- 写真は PhotoBlob（09）。カードのアイキャッチは Visit.eyecatchPath。

## 5. 要選択・未決事項（ユーザー判断）

1. **集計の重複**: 「今月の体験（4カウンター）」と「今月の記録（カテゴリ別）」を両方出すか、片方に統合するか。→ モックは両方併記。おすすめは **YOUR JOURNEY の直後に片方**（体験の物語 → 数字は1ブロックに集約）。
2. **YOUR JOURNEY の対象**: 直近1件の体験を追う（モック）か、複数体験の動きを時系列マージするか（案3は後者寄り）。
3. **ヒーローの見せ方**: 案3の白カード型（上品）か、案1の暗色大判キービジュ型（没入・テンション高）か。→ 「テンション爆上がる」重視なら案1寄せも検討。
4. **セリフ書体の具体**: 和文明朝＋欧文Didoneの銘柄選定（ライセンス含む・実装時）。
5. **サブコピー**: 「好きな体験を、人生の軌跡に。」（案3）／「あなたの『好き』を、記録に残そう。」（案1）のどちらを採用するか。正式キャッチは「観た・行った・体験したを、美しく一生残す。」（concept 確定）なので、ホームのサブは短い方を。
