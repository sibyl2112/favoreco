# ②-A / spec-A5 集計・カレンダー・地図・シリーズビュー

> ステータス: 清書（2026-07-06）
> 位置づけ: ②-A の「見せ方」分冊。記録（Event/Visit）から**統計・リピート・シリーズ・カレンダー・地図・フィルモグラフィ・味覚マップ・積読リスト**をどう導出・表示するかを定める。
> 関連: A1（Event/Visit・格納）／ A3（フィールド）／ A4（プリセット）／ 08 §4-3・feasibility §4-3（カレンダー確定ルール）／ feasibility §4-1c（酒蔵マップ）／ §4-1f（横断リンク）

---

## 0. 原則

- **集計は派生**：保存せず、クエリ可能な個別プロパティ（A1 §8-1）から `FetchDescriptor`＋Swift集計で都度算出（必要ならメモリキャッシュ）。unitFieldsRaw（表示専用）は集計に使わない。
- **同じユニット＝同じ統計**：同ユニットを使うカテゴリは統計が自動で揃い、**カテゴリ横断統計**（「今年の全体験」）が自然に出る（08 §1原則3）。
- 年間まとめ・月次リキャップ（課金機能）はこれらの集計の束ね。

---

## 1. 統計・集計（クエリ可能フィールドが源）

| 統計 | 源フィールド | 例 |
|---|---|---|
| 総合評価の分布・平均 | Visit.overallRating（Double 0.5刻み） | カテゴリ別/全体の★平均・高評価一覧 |
| ジャンル別 | Event.genres[String]（U15） | 「SF作品 n本」「今年ミステリー n」 |
| 評価軸の自分平均 | AxisScore（U7・Visit） | 観劇の"脚本"軸の平均・レーダー重ね |
| 金額集計 | Visit.money（U12）＋CollectionItem.amount（U11任意） | 年間支出・カテゴリ別・グッズ代・交通費・宿泊費・遠征費 |
| 期間・頻度 | Visit.date | 月別本数・年間タイムライン・皆勤 |
| 形態別 | Visit.engagementFormat / Event.workType（U14） | 劇場 vs 配信の比率・マンガ/技術書比率 |
| 種別別 | Event.subTypeKey（A2） | 酒種別内訳・施設種別内訳 |
| 当選率（チケット） | TicketAttempt.status(won/lost)×entryRoute×ticketSite/account | 先行区分別・販路別・名義別の当選率（spec-A8 §5・落選を統計に含める設定時） |
| セトリ演奏回数 | Visit.setlist（U4）横断・曲名正規化 | 「この曲を何回聴いた」（LiveSoul/Live Rock準拠・spec-A8 §5） |
| 名義別（FC/チケットアカウント） | TicketAttempt.account / Visitにコピーされた観劇実績 | 名義ごとの申込数・当選率・参戦数・年会費合計（掛け持ち可視化・spec-A8 §3.2） |
| 座席傾向 | Visit.seat（U2）×会場 | 会場別の座席分布（3D表示はspec-A8 §6でv2） |

- **カテゴリ横断「今年の全体験」**：全Visitを日付で束ね、`categoryRef`でグループ表示（Mystoriumはカテゴリ内のみ＝favorecoの新機能・feasibility §3）。
- **ヒートマップ（体験頻度暦）★2026-07-06**：日/週ごとの体験件数を色の濃淡で（GitHub草グラフ的）。**年間まとめ（買い切り）の顔**・カテゴリ横断。「今年これだけ体験した」を一望（Letterboxd/Untappd等のwrap文化の私的版・spec-B1 §9）。

## 2. リピートグループ（U13）

- **定義**：同一 Event 配下の Visit 群。保存フィールドを持たず**Event→Visit(多)の集計**（通算n回）。
- **表示**：対象カード「通算n回」＋回ごと比較（日付・会場・評価の推移）。代表アイキャッチ＝Event（A1 §5）、各回＝Visit.eyecatch。
- **同一対象の判定（名寄せ）はA6**（Event単位が基本・手入力は正規化upsert）。

---

## 3. シリーズビュー（書籍・2026-07-06確定／汎用化可）

- **グルーピング**：`Event.seriesName` 一致＝1シリーズ、`volumeNumber` で整列（A3）。ハードなSeriesエンティティは持たない（派生・v1）。
- **巻スクロール**：シリーズを開くと巻がvolumeNumber順に連続スクロール＝シリーズビュー。
- **一覧のスタック表示**：書籍一覧でシリーズを**背表紙/カバーを重ねた1タイル**に畳む（長編漫画がN個で埋まらない）。タップで展開・フラット/スタック切替。
- **最新刊登録(add-next)**：最新登録巻から属性（seriesName/著者/出版社/形態/ジャンル）を引き継ぎ**巻数+1**で新規（A7登録フロー）。
- 汎用化：seriesName はツアー名等にも使え、他カテゴリの「まとめ表示」に転用余地。

## 4. カレンダー（データ→ビューの束ね）

> 描き分けの確定ルールは feasibility §4-3・08 §4-3 が正本。A5はどのフィールドが何を描くかの対応を定める。

| ビュー要素 | 源 |
|---|---|
| 月表示の日セルのチップ（カテゴリ色） | Visit.date＋categoryRef.colorHex |
| 週/日の時間グリッドのブロック高さ | 開演＋上演時間→終演（U3・A3） |
| 期間バー（会期/販売期間・8px・白抜きタイトル・頭アイコン） | Event.periodStart/End＋種別/カテゴリ色・チケット販売期間 |
| 終日スロット（グレー枠線） | 終日フラグの体験（フェス全日・遠征） |
| 発売点チップ「発売 10:00」＋アラーム | チケット発売日時（U・観劇/美術展/映画） |
| 天気アイコン（日付の横） | Visit.weatherSymbol（過去日・出かける系・08 §4-9） |
| 記録レイヤー（済みの薄チップ✓） | 参加済みVisit |
| 外部カレンダー（低彩度・破線） | EventKit読み取り（Apple/Google・feasibility §4-3 B） |

- **レイヤーon/off**：①カテゴリ ②種別（予定/記録済み/チケット販売） ③外部カレンダー。
- ビュー切替（月/週/日）は年月タイトルの左（縦幅節約）。月グリッドは7列完全均等。

## 5. 地図（都道府県・訪問・汎用）

- **汎用マップ機能**：`PlaceMaster.地域/所在県` を集計し日本地図を県別に塗り分け（MapKit・meguri型応用・feasibility §4-1c）。種別×地域で色分け。
- **具体マップ**：
  - 酒蔵所在県マップ（Brewery.所在県・「どこの酒が好きか」）★酒キラー
  - 御朱印の都道府県マップ（集めた地域・王者も定番）
  - 訪問施設マップ（おでかけ施設）
  - 将来：城/寄港マップ（御城印/御船印）
- 所在県は列挙（47都道府県）で軽量。座標はPlaceMaster（A3）。

## 6. 人物集計・フィルモグラフィ（Person）

- **源**：Event↔Person credits（役割：監督/脚本/出演/作家・A3）。
- **表示**：「**この監督の映画 n本**」「この俳優が出てる n本」「この作家 n回」＝公開DBでなく**自分の鑑賞史**（favorecoらしい差別化・08 §4-5）。人物ページ＝その人に紐づくEvent一覧。

---

## 7. 味覚マップ集計（酒）

- **源**：Visit.味覚2軸(x,y)（U17・甘辛×濃淡）。
- **表示**：自分が飲んだ酒の**散布/密度**を2軸マップに重ね、**好みの中心**を可視化。さけのわのフレーバー数値は各銘柄の参考デフォルト。U7 6軸（さけのわ準拠）はオプションの重ねレーダー。

## 8. 読書状態ビュー（積読リスト）★A5でモデル修正

- **修正（2026-07-06）**：読書状態（読了/積読/中断/再読）は**本(Event)の現在の状態**（Event.readingStatus）に置く。理由＝**積読＝まだ読んでいない＝読んだ回(Visit)が存在しない**状態なので、Visit所属では表現できない。A3/A4を修正。
- **表示**：**積読リスト**（readingStatus=積読のEvent一覧）・読了数・中断本。再読は読んだVisitを足す（Event.readingStatusは"再読"や最新へ更新）。
- 基本ステータス（気になる/予定/済み・日付から自動推測）の書籍特化版。

---

## 9. 実装・パフォーマンスメモ

- 集計は個別プロパティへの`#Predicate`/FetchDescriptor＋Swift集約。件数増でも列インデックス的に軽い範囲。重い横断集計はメモリキャッシュ＋記録変更で無効化。
- サムネは非永続（09・AsyncImageDataViewダウンサンプリング）。スタック/巻スクロール/マップも遅延デコードで軽量維持。
- 外部カレンダー（EventKit）・天気（WeatherKit）はビュー表示時に非同期取得＋キャッシュ。

---

## 10. 持ち越し（A6）

- 同一対象判定（名寄せ）と共有マスター横断リンク（酒蔵ハブ・御酒印↔酒）の実装詳細。
- スナップショットの実装（マスター削除・参照DB消滅時の表示保持）。
- readingStatus のEvent移設に伴うA3/A4の追随（本コミットで反映）。
